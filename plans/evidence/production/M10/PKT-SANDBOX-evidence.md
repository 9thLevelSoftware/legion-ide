## Concerns

1. **Windows enforcement is weak.** The job-object-only path does not restrict filesystem writes or network access. The `SandboxEnforcementReport` is honest (`filesystem_write_enforced: false`, `network_enforced: false`), and caveat labels document the limitation. Upgrading to restricted-token or AppContainer would require `SE_ASSIGNPRIMARYTOKEN_PRIVILEGE`, which is unavailable to normal user processes. This is a known platform limitation, not an implementation defect.

2. **RUSTSEC-2026-0204 (crossbeam-epoch 0.9.18).** Pre-existing advisory in the workspace (via rayon → tantivy and wasmtime). Added to `deny.toml` ignore list. Not introduced by this packet. Fix requires upstream tantivy/wasmtime to accept `crossbeam-epoch >= 0.9.20`.

3. **Linux/macOS tests not exercised locally.** The Landlock and Seatbelt code paths are cfg-gated; they compile and logic is correct but were not exercised end-to-end in this Windows-only development environment. CI on Linux/macOS will be the first real exercise of those paths.
