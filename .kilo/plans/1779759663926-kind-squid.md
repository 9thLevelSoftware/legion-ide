Status: execution plan

## 1. Current State

Phase 8 is not complete and must remain `Not accepted` until this plan is fully executed.

Current accepted baseline:

| Area | Current state | Completion gap |
| --- | --- | --- |
| Remote transport | `devil-remote-transport` fixture validates metadata-only handshakes and frame descriptors. | No production network carrier, TLS/mTLS or equivalent, endpoint policy runtime, agent packaging, reconnect/resume, flow control, or platform evidence. |
| Terminal | `devil-terminal` fixture validates launch policy and bounded redacted output. | No native PTY lifecycle, input/resize/close/kill, process cleanup, platform backend, app composition, or platform evidence. |

Non-negotiable boundaries:

| Boundary | Required preservation |
| --- | --- |
| Projection-only UI | `devil-ui` only renders projections and emits intents. It must not own PTYs, transports, terminal sessions, storage, editor text, or mutation authority. |
| Proposal-mediated mutation | Remote, terminal, retention, telemetry, diagnostics, plugin, AI, and future surfaces cannot mutate workspace/editor/disk directly. File/editor changes must route through proposal/workspace/editor authorities and existing preconditions. |
| Default denial | Production transport, PTY, hosted telemetry, raw-source capture, and hosted raw bundle upload remain disabled until policy, capability, consent, and evidence gates enable them. |
| Air-gap authority | Air-gap mode denies non-loopback network transport, hosted telemetry, hosted raw-source upload, update checks, and hosted providers. |
| Metadata-only default | Normal telemetry, observability, storage, remote, terminal, plugin, AI, collaboration, and diagnostics records must reject raw source, terminal transcripts, process output, transport payload bodies, prompts, provider payloads, and secrets. |
| Evidence-first acceptance | Phase 8 may be marked accepted only after all required artifacts exist, final checklist items are checked, full validation commands pass, and `xtask check-deps` accepts the governance state. |

## 2. Execution Strategy

Implement Phase 8 as a sequence of gated slices. Each slice must leave the repository in a passing, safe state. Runtime surfaces should move through three stages: protocol and policy contracts, production-capable implementation behind explicit feature/policy gates, then production acceptance after evidence is archived.

Recommended gate order:

| Gate | Focus | Exit criteria |
| --- | --- | --- |
| G0 | Governance lock | ADR-0025 through ADR-0029 are rewritten for production decisions, owners are recorded, threat model outline exists, and dependency deltas are approved but Phase 8 remains not accepted. |
| G1 | Protocol and policy contracts | `devil-protocol`, `devil-security`, `plans/dependency-policy.md`, and `xtask` describe every active production contract before runtime code depends on it. |
| G2 | Default-deny integration | Security tests prove all Phase 8 capabilities deny by default, deny in air-gap where required, require workspace trust/principal/capability/consent, and reject raw payload persistence. |
| G3 | Remote transport production core | Secure typed-envelope transport, endpoint policy, identity binding, resume/flow control, agent packaging checks, and metadata audit pass tests behind explicit activation. |
| G4 | Native terminal/PTTY runtime | Terminal state machine, app composition, PTY backend, lifecycle actions, output projection, cleanup, and platform evidence pass. |
| G5 | Hosted telemetry operations | Durable metadata spool, classifier, consent/revoke, hosted exporter, retry/drop/backpressure, and egress controls pass. |
| G6 | Encrypted raw-source vault | Scoped consent, encrypted local vault, key references, TTL, access audit, delete/revoke/tombstone, backup/restore, and corruption handling pass. |
| G7 | Migrations and diagnostics | Migration registry, dry-run, backup/recover, read-only diagnostics, metadata replay, subsystem health, and enterprise profile CI pass. |
| G8 | Fault, performance, platform | Fault drills, p50/p95/p99 budgets, Windows 11 plus at least one Unix-like platform matrix, and cargo-deny production dependency review pass. |
| G9 | Release candidate | Evidence artifacts are complete, full workspace gates are archived, rollback/incident/canary plans are signed off, and residual risks have owners/dates. |
| G10 | GA acceptance flip | ADR statuses, phase ledger, architecture map, and final checklist are updated to accepted only after G0-G9 pass and `xtask check-deps` validates acceptance governance. |
