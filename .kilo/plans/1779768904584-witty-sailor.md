## Non-Negotiable Boundaries

- `devil-ui` remains projection-only. It may emit intents and render projections, but it must not own terminal sessions, PTYs, transports, telemetry spools, retention vaults, storage migrations, or mutation authority.
- Phase 8 runtime crates must not mutate editor/workspace/disk directly outside accepted proposal/workspace authority.
- Remote production transport must reject plaintext and unauthenticated production carrier paths.
- Terminal GA cannot count process-backed degraded mode as native PTY parity.
- Hosted telemetry cannot export without explicit current consent and cannot attempt DNS/network in air-gap denial cases.
- Raw-source production config must not instantiate XOR or deterministic local cipher behavior.
- Metadata, telemetry, audit, diagnostics, and evidence must never include raw source, terminal output, transport payloads, prompts, provider payloads, secrets, tokens, or key bytes.
- Acceptance docs must not flip before refreshed full gate outputs are archived under `plans/evidence/phase-8/`.

## Execution Model

Implement in gated slices. Each gate should leave the repository compiling and default-deny safe. Run focused tests at the end of each slice; run full gates before the final acceptance flip.

## Stop And Rollback Conditions

Stop implementation and roll back the active slice if any of these occur:

- UI owns terminal/remote/telemetry/retention state.
- A Phase 8 runtime mutates editor/workspace/disk directly outside proposal/workspace authority.
- Remote carrier accepts plaintext or unauthenticated production transport.
- Terminal GA path falls back to process-backed degraded mode.
- Hosted telemetry exports without explicit consent or in air-gap mode.
- Raw-source vault can be configured with XOR/deterministic cipher in production.
- Raw source, terminal output, transport payloads, prompts, provider payloads, secrets, tokens, or key bytes appear in metadata, telemetry, audit, diagnostics, or evidence.
- Acceptance docs are flipped before current full gate outputs are archived.

## Practical Implementation Notes

- Prefer minimal crate-local modules first, then extract only where reuse is real.
- Keep fixture runtimes for deterministic tests, but make production paths explicit and impossible to confuse with GA behavior.
- Add DTOs and policy gates before network/process/crypto runtime code so failures are validator/security errors, not ad hoc runtime checks.
- Every new dependency must be reflected in `plans/dependency-policy.md`, workspace manifests, `xtask`, and `deny.toml` review before relying on it.
- Avoid adding backward-compatibility shims unless persisted data migration requires them.
- Run focused tests for each slice before moving on; run full gates only after broad integration points settle.
