## Non-Negotiable Boundaries

| Boundary | Required behavior |
| --- | --- |
| UI projection-only | `devil-ui` may emit `CommandDispatchIntent` and render projections only. It must not own transports, PTYs, terminal sessions, telemetry spools, retention vaults, storage migrations, or mutation authority. |
| Proposal-mediated mutation | Remote, terminal, telemetry, retention, diagnostics, plugins, AI, and collaboration surfaces must not directly mutate editor/workspace/disk state. File/editor changes must route through existing proposal/workspace/editor authorities and preconditions. |
| Default-deny activation | Production transport, native PTY, hosted telemetry, raw-source capture/export, and migration repair remain disabled unless ADR, policy, consent/capability, runtime, tests, and evidence gates are satisfied. |
| Air-gap safety | Air-gap mode denies non-loopback remote transport, hosted telemetry, hosted raw-source export, update checks, and hosted providers. |
| Metadata-only defaults | Observability, telemetry, storage, transport audits, terminal audits, diagnostics, plugin, AI, collaboration, and remote metadata must reject raw source, terminal transcripts, process output, transport payload bodies, prompts, provider payloads, secrets, and full environment values by default. |
| Evidence-first acceptance | Do not mark Phase 8 accepted until all required artifacts exist, checklist items are checked, full validation commands pass, and `cargo run -p xtask -- check-deps` passes after the acceptance flip. |

## Execution Strategy

Implement Phase 8 as gated slices. Each gate must leave the repository in a passing, safe state. Runtime crates should move from fixture contracts to production-capable implementations behind explicit policy/feature gates, then to accepted GA only after evidence is archived.

Recommended gate order:

| Gate | Focus | Exit criteria |
| --- | --- | --- |
| G0 | Governance and scope lock | ADR-0025 through ADR-0029 define accepted production scope, owners, dependencies, residual reservations, and stop conditions. Phase 8 remains not accepted. |
| G1 | Protocol and policy contracts | `devil-protocol`, `devil-security`, dependency policy, and `xtask` define required contracts before runtime implementation depends on them. |
| G2 | Default-deny integration | Security tests prove all Phase 8 production capabilities deny by default, deny in air-gap where required, require trusted workspace/principal/capability/consent, and reject raw payload persistence. |
| G3 | Production remote transport | Secure typed-envelope transport, endpoint policy, identity binding, reconnect/resume, flow control, replay defense, and agent packaging checks pass behind explicit activation. |
| G4 | Native terminal/PTTY runtime | PTY lifecycle, input/resize/close/kill, policy enforcement, output projection, cleanup/orphan handling, and platform evidence pass. |
| G5 | Hosted telemetry | Durable bounded metadata spool, classifier, consent/revoke, hosted exporter, retry/drop/backpressure, and egress controls pass. |
| G6 | Encrypted raw-source vault | Scoped consent, encrypted local vault, key references, TTL, access audit, delete/revoke/tombstone, backup/restore, and corruption handling pass. |
| G7 | Storage migrations and diagnostics | Explicit migration registry, dry-run, backup/checksum/recovery, read-only diagnostics, metadata replay, and enterprise policy profile CI pass. |
| G8 | Fault, performance, platform, supply chain | Fault drills, performance budgets, claimed platform matrix, and cargo-deny review pass and are archived. |
| G9 | Release readiness | Evidence artifacts are complete; rollback, canary, incident, support/debug-bundle, and residual risk signoffs are recorded. |
| G10 | Final acceptance flip | ADR statuses, phase ledger, architecture map, and checklist are marked accepted only after G0-G9 pass and final gates are archived. |

## Stop Conditions

Do not mark Phase 8 accepted if any condition below is true:

| Stop condition | Required action |
| --- | --- |
| Any runtime mutates workspace/editor/disk outside proposal/workspace/editor authorities. | Stop, fix boundary, add ownership regression tests. |
| Any raw source, transcript, process output, transport payload, prompt, provider payload, secret, or full env value enters metadata-only storage/telemetry/observability. | Stop, fix classifier/schema/storage path, expand adversarial tests. |
| Non-loopback egress succeeds under air-gap. | Stop, fix security broker and transport/exporter policy. |
| Native PTY leaves orphan processes after close, kill, timeout, crash, or app shutdown. | Stop GA for that platform or fail closed until fixed. |
| Migration cannot recover from interruption/corruption. | Stop, fix migration/recovery framework. |
| Raw-source vault cannot prove encryption, access audit, deletion, TTL expiry, backup/restore, and key-loss behavior. | Stop, fix vault lifecycle and evidence. |
| Cargo-deny reports unreviewed production network/crypto/process/PTY dependency risk. | Stop until security review or dependency removal. |
| Evidence claims a platform or capability without passing tests or documented drill evidence. | Remove claim or block GA. |
| `xtask check-deps` fails after the acceptance flip. | Revert acceptance text or complete missing evidence/governance work. |

## Completion Definition

Phase 8 is complete only when all of these are true:

- Production remote network transport is implemented, policy-gated, typed-envelope-only, secure, resumable, flow-controlled, metadata-audited, agent-package-aware, and proposal-bound.
- Native terminal/PTTY runtime is implemented, app-composed, capability-gated, bounded, redacted, cleanup-safe, platform-evidenced, and proposal-bound for mutations.
- Hosted telemetry export and durable spool are implemented, opt-in, air-gap denied, metadata-only, consent-aware, classified, resilient, diagnosable, and non-blocking.
- Encrypted raw-source vault is implemented, default-deny, explicit-consent, purpose-bound, TTL-bound, access-audited, deletable, recoverable, and isolated from normal metadata paths.
- Migration/recovery framework covers all active stores with manifests, dry-run, backups, checksums, interruption recovery, corruption quarantine, explicit repair flags, and replay drills.
- Fault drills, performance evidence, platform matrix, enterprise policy CI, cargo-deny review, release readiness, rollback, incident, and canary evidence are archived.
- ADR-0025 through ADR-0029 are accepted for production scope.
- Full validation command outputs are archived under `plans/evidence/phase-8/`.
