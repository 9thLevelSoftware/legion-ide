# Plan 04-04: Desktop Language And Terminal Panels

## Objective

Render and route the Phase 4 language and terminal IDE loop in `devil-desktop` using only projection snapshots and command intents.

- evidence-before-action: confirm the exact `CommandDispatchIntent` variants and projection fields from Plan 04-01 before adding desktop actions.
- minimal diff: keep changes inside desktop view, bridge, workflow, and one focused desktop test file.
- verify-before-report: run every frontmatter command and record command outcomes in `04-04-RESULT.md`.
- role: desktop UI implementer with projection-boundary discipline.
- task: add visible language and terminal panels plus intent routing without owning app/runtime/editor state.
- scope: panel rows, controls, prompts, action mapping, status rendering, denial/error rendering, and desktop tests.
- allowed tools/actions: focused Rust/egui edits, desktop unit/integration tests, cargo check/test, and source-boundary greps.
- forbidden actions: importing index/terminal/security/editor/project/storage crates into desktop, bypassing app command dispatch, storing raw terminal transcript state, or directly mutating app internals.
- verification criteria: visible panels render all roadmap surfaces, actions become `CommandDispatchIntent` values, denial/error states are visible, and desktop does not depend on runtime crates.
- final result format: write `.planning/phases/04-language-and-terminal-ide-loop/04-04-RESULT.md` with implementation summary, command outcomes, and unresolved risks.

</execution_contract>

## Task 1: Render Language Tooling Projection Rows

### Read Targets

- `crates/devil-desktop/src/view.rs`
- `crates/devil-ui/src/ui.rs`
- `crates/devil-protocol/src/lib.rs`

### Write Targets

- `crates/devil-desktop/src/view.rs`

### Implementation Sequence

1. Add a compact language/problematics panel that renders diagnostics/problems, hover, completion, definition, references, outline, and operation status rows from projection data.
   > verification: `rg -q "render_language" crates/devil-desktop/src/view.rs`
2. Add visible rows/buttons for formatting, rename, organize imports, and code actions that show proposal status rather than implying direct mutation.
   > verification: `rg -q "organize" crates/devil-desktop/src/view.rs`
3. Render unsupported, denied, failed, cancelled, and stale statuses distinctly without panics.
   > verification: `rg -q "cancel" crates/devil-desktop/src/view.rs`

### Edge And Error Cases

- Empty projections render disabled/empty states.
- Long diagnostic or completion text is truncated/wrapped within existing panel constraints.
- No raw source text is added to desktop-owned state.

<verify>

```powershell

cargo check -p devil-desktop --all-targets
```

</verify>

## Task 2: Render Terminal Panel And Route Desktop Actions

### Read Targets

- `crates/devil-desktop/src/view.rs`
- `crates/devil-desktop/src/bridge.rs`
- `crates/devil-desktop/src/workflow.rs`
- `crates/devil-ui/src/ui.rs`

### Write Targets

- `crates/devil-desktop/src/view.rs`
- `crates/devil-desktop/src/bridge.rs`
- `crates/devil-desktop/src/workflow.rs`

### Implementation Sequence

1. Add a terminal panel that renders runtime status, policy decision, bounded output rows, search/scrollback status, and last denial/error state.
   > verification: `rg -q "render_terminal" crates/devil-desktop/src/view.rs`
2. Add `DesktopAction` variants and bridge mappings for terminal launch, input, resize, search, close, and kill, plus language action buttons where needed.
   > verification: `rg -q "TerminalLaunch" crates/devil-desktop/src/bridge.rs`
3. Add workflow prompt state for terminal launch/input/search and rename input if the command variants require user-entered text.
   > verification: `rg -q "terminal_prompt" crates/devil-desktop/src/workflow.rs`

### Edge And Error Cases

- Denied terminal launch remains visible until superseded by another terminal action.
- Resize action must be safe when no session is running.
- Kill/close buttons dispatch intents only; desktop does not inspect runtime handles.

<verify>

```powershell
rg -q "TerminalLaunch" crates/devil-desktop/src/bridge.rs
rg -q "terminal_prompt" crates/devil-desktop/src/workflow.rs
cargo check -p devil-desktop --all-targets
```

</verify>

### BLOCKED Conditions

- The app/UI contract lacks a necessary terminal command intent.
