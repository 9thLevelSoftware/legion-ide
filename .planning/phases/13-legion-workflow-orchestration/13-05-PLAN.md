## Task 1: Add App-Owned Workflow Session Lifecycle

### Read Targets

- `crates/devil-app/src/lib.rs`
- `crates/devil-agent/src/lib.rs`
- `crates/devil-protocol/src/lib.rs`
- `crates/devil-tracker/src/lib.rs`
- `crates/devil-memory/src/lib.rs`

### Write Targets

- `crates/devil-app/src/lib.rs`

### Implementation Sequence

1. Add app-owned storage for active Legion workflow sessions, bounded by metadata-only protocol DTOs. Filter out empty session ids and invalid schema/correlation data.
2. Add `seed_legion_workflow_sessions` for tests/smoke harnesses and `execute_legion_workflow` or equivalent method that starts a session through the `LegionWorkflowCoordinator`.
3. Route local worker proposal outputs into existing proposal/proposal-preview paths; provider-backed workers must produce assisted-AI route metadata or blocked state, not direct invocation.
4. Emit tracker ledger records and memory outcome candidates only with metadata and consent state.
   > verification: `rg -q "execute_legion_workflow" crates/devil-app/src/lib.rs`

### Edge And Error Cases

- Empty or duplicate session ids are rejected.
- Provider route metadata missing blocks.
- Worker dependency cycle or missing dependency blocks.
- Invalid tracker/memory record blocks rather than reporting success.

## Task 2: Add Verification, Sign-Off, Conflict, And Merge Readiness Routing

### Read Targets

- `crates/devil-app/src/lib.rs`
- `crates/devil-protocol/src/lib.rs`

### Write Targets

- `crates/devil-app/src/lib.rs`

### Implementation Sequence

1. Add methods to record verification evidence metadata, record reviewer sign-off metadata, resolve conflicts by metadata id, and compute merge readiness from protocol readiness evaluators.
2. Merge readiness must block for unresolved conflict, missing verification, failed/blocked verification, missing sign-off, stale proposal preconditions, dirty main workspace conflict, missing rollback/checkpoint metadata when required, or missing audit-before-success.
3. If merge readiness is accepted, produce or select a proposal/approval artifact for the merge path; do not perform git merge, file writes, editor mutation, or proposal apply.
4. Ensure all status labels are display-safe and redaction hints remain metadata-only.
   > verification: `rg -q "LegionWorkflow" crates/devil-app/src/lib.rs`

### Edge And Error Cases

- A completed worker with no linked proposal/evidence cannot advance merge readiness.
- A sign-off from an unknown reviewer/principal is recorded as denied or blocked.
- Conflict resolution without verification evidence remains blocked.
