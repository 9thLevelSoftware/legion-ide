# Plan 05-04: Assisted AI Explain And Propose Routing

## Objective

Add assisted-AI explain and propose flows that are local-first/default-deny, visibly refusal-aware, and proposal-only for generated edits.

- verify-before-report: run every frontmatter command and record outcomes in `05-04-RESULT.md`.
- role: AI workflow implementer with security/default-deny review.
- task: expose explain/propose flows, keep provider routing local-first/default-deny, surface refusal/redaction/budget data, and ensure proposed edits never self-apply.

## Task 1: Generalize App-Owned Assisted AI Runs

### Read Targets

- `crates/devil-app/src/lib.rs`
- `crates/devil-protocol/src/lib.rs`
- `crates/devil-app/tests/control_trust_surfaces.rs`

### Write Targets

- `crates/devil-app/src/lib.rs`
- `crates/devil-app/tests/control_trust_surfaces.rs`

### Implementation Sequence

1. Keep the existing `start_ai_run` public method as a propose-edit compatibility wrapper, but route it through a new internal `run_assisted_ai_operation(operation_class, instruction_label, provider_class)` helper that accepts `AssistedAiOperationClass`.
   > verification: `rg -q "AssistedAiOperationClass::Explain" crates/devil-app/src/lib.rs`
2. Update `AppAiRunOutcome` so it can represent both success and refusal without opaque errors: use `proposal_id: Option<ProposalId>`, `proposal_created: Option<ProposalResponse>`, keep `route_response`, `context_manifest_projection`, and `privacy_inspector_projection`, and add `refusal: Option<AssistedAiRefusalMetadata>`. Existing propose-edit tests must be updated to assert `Some(...)`.
   > verification: `rg -q "proposal_id: Option<ProposalId>" crates/devil-app/src/lib.rs`
3. Add an explain path that assembles the same context/privacy/budget projections but produces metadata-only explanation provenance and no proposal apply path.
   > verification: `cargo test -p devil-app --test control_trust_surfaces assisted_ai_explain -- --nocapture`
4. Keep propose-edit output as a `WorkspaceProposal` registered through the proposal coordinator; assert the active buffer and disk remain unchanged until explicit proposal apply.
   > verification: `cargo test -p devil-app --test control_trust_surfaces assisted_ai_propose_is_proposal_only -- --nocapture`

### Edge And Error Cases

- Explain may produce bounded labels/provenance only; do not store raw LLM answer text.
- Propose edit must include proposal preconditions and context/trust references.

<verify>

```powershell
cargo test -p devil-app --test control_trust_surfaces assisted_ai_explain -- --nocapture
cargo test -p devil-app --test control_trust_surfaces assisted_ai_propose_is_proposal_only -- --nocapture
```

</verify>

### BLOCKED Conditions

- Explain/propose cannot share app-owned trust assembly without duplicating raw context.

## Task 2: Surface Provider Refusals Instead Of Opaque Errors

### Read Targets

- `crates/devil-app/src/lib.rs`
- `crates/devil-ai/src/lib.rs`
- `crates/devil-ai-providers/src/lib.rs`
- `crates/devil-security/src/lib.rs`

### Write Targets

- `crates/devil-app/src/lib.rs`
- `crates/devil-ai/src/lib.rs`
- `crates/devil-ai-providers/src/lib.rs`
- `crates/devil-app/tests/control_trust_surfaces.rs`

### Implementation Sequence

1. Ensure local deterministic provider routes support explain and propose labels without cloud credentials.
   > verification: `cargo test -p devil-ai-providers --all-targets`
2. Ensure default-denied, untrusted, hosted/remote, local-provider-disabled, privacy-denied, and budget-denied routes return `AssistedAiRefusalMetadata` and update `AssistedAiProjection.refusals`.
   > verification: `cargo test -p devil-ai router -- --nocapture`
3. In app AI orchestration, persist refusal audit metadata and return an app outcome/projection that the shell can display instead of losing the refusal in an error string.
   > verification: `cargo test -p devil-app --test control_trust_surfaces assisted_ai_refusals_visible -- --nocapture`

### Edge And Error Cases

- Refusal paths must not create proposals.
- Refusal projections must include reason category, trust/privacy/budget labels, and redaction hints.
- Hosted/remote provider requests must remain denied by default.

cargo test -p devil-app --test control_trust_surfaces assisted_ai_refusals_visible -- --nocapture
