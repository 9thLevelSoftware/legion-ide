# Plan 07-02: End-To-End Local IDE Beta Smoke Harness

## Objective

Add deterministic beta workflow smoke coverage proving the local IDE can open, browse, edit, search, save, use language/terminal surfaces, and review proposal flows without mutating the user's checkout or bypassing app authority.

<execution_contract>

- read-before-write: read `07-CONTEXT.md`, `07-01-RESULT.md`, `AGENTS.md`, `crates/devil-desktop/src/workflow.rs`, `smoke.rs`, `view.rs`, `bridge.rs`, existing desktop workflow/search/language/terminal/control-trust tests, `scripts/gui-smoke.ps1`, `scripts/gui-smoke.sh`, `.github/workflows/ci.yml`, and `plans/evidence/gui-productization/phase-7-local-ide-beta.md` before editing.
- evidence-before-action: confirm existing desktop actions already route through `AppComposition` and `CommandDispatchIntent`; build the beta harness from those existing actions rather than adding new authority paths.
- minimal diff: add a beta smoke harness and wrapper options; do not edit app/UI/protocol contracts unless a stop gate fires.
- verify-before-report: run every frontmatter command and record outcomes in `07-02-RESULT.md`.
- role: local IDE workflow implementer and QA verifier.
- task: create an automated, metadata-only beta smoke path that exercises the critical local IDE workflow and writes evidence.
- scope: `devil-desktop` beta/smoke/workflow glue, tests, smoke wrappers, CI dry-run hooks, workflow evidence, and result file.
- allowed tools/actions: focused Rust/script/YAML/Markdown edits, temporary workspace creation under `target/`, `cargo test`, smoke runs, and `rg`.
- forbidden actions: editing `devil-app`, `devil-ui`, or protocol authority contracts; writing into source files in the real checkout during smoke; adding dependencies; claiming unsupported remote/collaboration/plugin/autonomy readiness.
- verification criteria: beta smoke edits only the isolated `--beta-workspace`, preserves projection-only/authority boundaries, writes metadata-only evidence, and keeps Phase 6 smoke wrapper defaults backward compatible.
- final result format: write `.planning/phases/07-fully-functional-local-ide-beta/07-02-RESULT.md` with changed files, command outcomes, smoke status, and any `BLOCKED` items.

</execution_contract>

<stop_gates>

- BLOCKED if beta workflow smoke requires direct file mutation outside `DesktopRuntime`/`AppComposition` authority.
- BLOCKED if the harness cannot isolate write actions from the user's actual checkout.
- BLOCKED if terminal or language tooling evidence can only be produced by claiming direct mutation or unsupported live LSP/PTTY behavior.
- BLOCKED if smoke evidence would include raw source bodies, dirty buffer text, terminal payloads, provider payloads, prompts, or secrets.

</stop_gates>

<recovery>

- If `--beta-smoke` argument parsing conflicts with existing `--smoke`, keep existing `--smoke` behavior unchanged and make beta mode an explicit mutually exclusive flag with a clear parse error when combined incorrectly.
- If native-window smoke is blocked on the local host, keep the beta workflow harness testable without a native window and record the native blocker separately in evidence.
- If a beta action fails because an app surface is intentionally default-deny, record the denial as expected metadata evidence rather than changing policy.
- On interruption, inspect `git diff`, compare modified files against `files_modified` and `files_forbidden`, rerun verification, and continue only from unverified work.

</recovery>

<verification>

- [ ] `crates/devil-desktop/src/beta.rs` creates or refreshes only `target/gui-phase7-beta-workspace` for write actions.
- [ ] Beta workflow opens a Rust workspace, browses/refreshes explorer, edits/saves a Rust file, runs active-file and workspace search, triggers language tooling, triggers terminal denial/bounded status, and creates/previews a proposal.
- [ ] `scripts/gui-smoke.ps1` and `scripts/gui-smoke.sh` retain Phase 6 defaults unless beta mode is explicitly requested.
- [ ] CI contains non-interactive GUI Phase 7 dry-run checks.
- [ ] `plans/evidence/gui-productization/phase-7-local-workflow-smoke.md` contains metadata-only status and no raw source.
- [ ] All frontmatter verification commands pass or the plan writes `BLOCKED` with exact evidence.

</verification>

- The implementation does not expand app/UI/protocol authority or introduce dependencies.

</success_criteria>

## Task 1: Add Beta Workflow Harness

### Read Targets

### Write Targets

- `crates/devil-desktop/src/beta.rs`
- `crates/devil-desktop/src/lib.rs`
- `crates/devil-desktop/src/workflow.rs`
- `crates/devil-desktop/src/smoke.rs`
- `crates/devil-desktop/tests/beta_workflow.rs`

### Implementation Sequence

1. Add `crates/devil-desktop/src/beta.rs` with `BetaWorkflowConfig`, `BetaWorkflowReport`, and `run_beta_workflow(config)`. The config must include `real_workspace_root`, `beta_workspace_root`, `evidence_path`, `session_state`, and `diagnostics_export`.
   > verification: `test -f crates/devil-desktop/src/beta.rs`
2. Implement `prepare_beta_workspace` to create `target/gui-phase7-beta-workspace` as a minimal Rust workspace with `Cargo.toml`, `src/main.rs`, and `README.md`. If the directory exists, replace only files under that exact directory after verifying the resolved path is inside `target/`.
   > verification: `rg -q "gui-phase7-beta-workspace" crates/devil-desktop/src/beta.rs`
3. Run beta actions through `DesktopRuntime::handle_action`: refresh/toggle explorer, open `src/main.rs`, insert text, save active buffer, run active-file search, run workspace search, request completion/cancel language operation, terminal launch denial or bounded terminal status, start AI proposal, open/preview proposal details, and quit.
   > verification: `rg -q "DesktopAction::StartAiProposal" crates/devil-desktop/src/beta.rs`
4. Render `BetaWorkflowReport::to_markdown()` with counts/status labels only: opened workspace, browsed files, edit/save result, search result counts, language status label, terminal policy/denial label, proposal lifecycle label, unsupported-surface labels, diagnostics path, and errors. Do not include source text or command payloads.
   > verification: `rg -q "unsupported_surfaces" crates/devil-desktop/src/beta.rs`
5. Add `--beta-smoke` and `--beta-workspace` parsing to `DesktopLaunchConfig` or a sibling launch mode without changing current `--smoke` behavior. Route beta mode from `run_from_env`.
   > verification: `cargo test -p devil-desktop --test beta_workflow -- --nocapture`

### Edge And Error Cases

- If `--beta-smoke` and `--smoke` are both supplied, return a parse error.
- If `--beta-workspace` resolves outside `target/`, return an error.
- If save is rejected, keep the buffer dirty and record `save_rejected` rather than pretending success.
- If terminal launch is denied by policy, record `terminal_denied_expected` as beta proof.
- If language mutation action creates a proposal preview instead of applying edits, record the proposal id/lifecycle only.

```powershell
rg -q "unsupported_surfaces" crates/devil-desktop/src/beta.rs
cargo test -p devil-desktop --test beta_workflow -- --nocapture
```

</verify>

### BLOCKED Conditions

- The workflow cannot be exercised through existing `DesktopAction` and app-owned outcomes.
