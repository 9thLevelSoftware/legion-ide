# Plan 07-04: Beta Launch Docs Known Limitations And Release Readiness

## Objective

Document how to launch and evaluate the local IDE beta, record known limitations honestly, and prepare beta readiness evidence without claiming unsupported advanced GUI behavior.

<execution_contract>

- read-before-write: read `07-CONTEXT.md`, `07-02-RESULT.md`, `07-03-RESULT.md`, `AGENTS.md`, `plans/evidence/gui-productization/phase-7-local-ide-beta.md`, `phase-7-local-workflow-smoke.md`, `phase-7-operational-health-diagnostics.md`, Phase 6 package runbook, and current scripts before writing docs.
- evidence-before-action: base every launch/readiness claim on commands or artifacts that exist after Plans 07-02 and 07-03.
- minimal diff: create only Phase 7 GUI productization documentation/evidence files and this result.
- verify-before-report: run every frontmatter command and record outcomes in `07-04-RESULT.md`.
- role: beta documentation and release-readiness coordinator.
- task: produce launch docs, known limitations, manual evidence, and release-readiness checklist for the local IDE beta.
- scope: Markdown docs under `plans/evidence/gui-productization/` and result file.
- allowed tools/actions: Markdown edits, `rg`, reading current scripts/evidence, and no code changes.
- forbidden actions: code/script/CI/gate changes, editing legacy Phase 7 remote evidence, marking acceptance final, or adding unsupported claims.
- verification criteria: docs include exact commands, common workflows, manual evidence fields, limitation labels, and readiness checks.
- final result format: write `.planning/phases/07-fully-functional-local-ide-beta/07-04-RESULT.md` with changed files, documentation checks, and any `BLOCKED` items.

</execution_contract>

<stop_gates>

- BLOCKED if launch docs would need commands that do not exist after Plans 07-02 and 07-03.
- BLOCKED if the known limitations cannot honestly separate local beta support from remote/collaboration/plugin/autonomy/GA scope.
- BLOCKED if release-readiness signoff would require evidence not produced by this phase.

</stop_gates>

<recovery>

- If a Plan 07-02 or 07-03 artifact is missing, keep the relevant doc section as `Not accepted` and record the blocker instead of inventing evidence.
- If a limitation is ambiguous, write it as unsupported for GUI Phase 7 until a later phase explicitly accepts it.
- If launch commands change during build, update docs from live script output and rerun `rg` checks.
- On interruption, inspect `git diff`, compare modified files against `files_modified` and `files_forbidden`, rerun verification, and continue only from unverified work.

</recovery>

<verification>

- [ ] Launch runbook includes exact normal launch, package dry-run, beta smoke, diagnostics export, and evidence check commands.
- [ ] Known limitations explicitly say remote production GUI, collaboration GUI, plugin marketplace/management, hosted provider activation, autonomous apply, signed installer, and platform parity are unsupported by GUI Phase 7 unless later accepted.
