# Plan 06-05: GUI Smoke Scripts And CI Coverage

## Objective

Add repeatable GUI smoke scripts and initial CI dry-run coverage for package, launch, workflow, platform, diagnostics, and evidence checks.

<execution_contract>

- read-before-write: read `06-CONTEXT.md`, `06-04-RESULT.md`, `scripts/package-windows.ps1`, `scripts/run-phase-gates.ps1`, `scripts/run-phase-gates.sh`, `.github/workflows/ci.yml`, `crates/devil-cli/src/main.rs`, and `plans/evidence/gui-productization/phase-6-package-runbook.md` before editing.
- evidence-before-action: confirm current scripts do not cover GUI smoke and current CLI evidence phases do not include GUI Phase 6.
- minimal diff: edit only scripts, CI workflow, CLI evidence checker, parity plan, and result.
- verify-before-report: run both smoke scripts in dry-run mode and CLI tests; record outcomes in `06-05-RESULT.md`.
- role: CI/smoke automation implementer with workflow QA.
- task: create smoke scripts and evidence checks covering install/package, launch, open workspace, edit/save, terminal, LSP/language, proposal review, diagnostics export, and quit.
- scope: scripts, CI dry-run step, CLI evidence check, parity plan, and result.
- allowed tools/actions: script/CLI/CI edits, dry-run script execution, `cargo test -p devil-cli`, and `rg`.
- forbidden actions: changing desktop implementation, requiring interactive GUI in CI, weakening existing CI gates, adding dependencies, or editing manifests.
- verification criteria: dry-run scripts exit zero, CLI evidence check tests pass, CI includes non-interactive GUI Phase 6 smoke step, and parity plan documents Windows/macOS/Linux behavior honestly.
- final result format: write `.planning/phases/06-packaging-platform-integration-and-accessibility/06-05-RESULT.md` with changed files, command outcomes, and blockers.

</execution_contract>

<stop_gates>

- BLOCKED if smoke scripts would run destructive package cleanup or write outside `-OutDir`.
- BLOCKED if CI requires opening an interactive native window on Linux/macOS without a headless strategy.
- BLOCKED if CLI evidence check duplicates `xtask` in a contradictory way.
- BLOCKED if scripts cannot cover terminal/language/proposal paths at least through deterministic desktop workflow/test commands.

</stop_gates>

<recovery>

- If full GUI launch is not viable in CI, keep CI at dry-run plus targeted desktop tests and document manual Windows smoke in the parity plan.
- If shell portability becomes brittle, keep PowerShell as authoritative for Windows and POSIX as dry-run parity only.
- If `devil-cli` evidence check scope grows too large, validate only the GUI Phase 6 evidence file and required artifact presence; leave final gate enforcement to `xtask`.
- On interruption, inspect `git diff`, compare modified files against `files_modified` and `files_forbidden`, rerun verification, and continue only from unverified work.

</recovery>

## Task 1: Add GUI Smoke Scripts

### Read Targets

- `scripts/package-windows.ps1`
- `scripts/run-phase-gates.ps1`
- `scripts/run-phase-gates.sh`
- `crates/devil-desktop/src/workflow.rs`

### Write Targets

- `scripts/gui-smoke.ps1`
- `scripts/gui-smoke.sh`

### Implementation Sequence

1. Create `scripts/gui-smoke.ps1` with parameters `-Workspace`, `-OutDir`, `[switch] -DryRun`, and optional `-DurationMs`. It must resolve paths safely, create `-OutDir` only when not dry-run, and call or print package, desktop smoke, diagnostics export, and targeted workflow test commands.
   > verification: `rg -q "GUI Phase 6 smoke" scripts/gui-smoke.ps1`
2. Create `scripts/gui-smoke.sh` with equivalent `--workspace`, `--out-dir`, `--dry-run`, and `--duration-ms` options using POSIX-safe path handling and no recursive deletion.
   > verification: `rg -q "GUI Phase 6 smoke" scripts/gui-smoke.sh`
3. Include command coverage labels for package/install, launch, open workspace, edit/save, terminal, language/LSP, proposal review, diagnostics export, and quit. Use existing targeted tests where direct UI automation is unavailable.
   > verification: `pwsh -NoProfile -File scripts/gui-smoke.ps1 -Workspace . -OutDir target/devil-gui-smoke/phase6-dry-run -DryRun`

### Edge And Error Cases

- Scripts must fail if `-OutDir` or `--out-dir` resolves to the workspace root.
- Dry-run must not write evidence files.
- POSIX script must not assume Windows `.exe` exists.

<verify>

```powershell
rg -q "GUI Phase 6 smoke" scripts/gui-smoke.ps1
rg -q "GUI Phase 6 smoke" scripts/gui-smoke.sh
pwsh -NoProfile -File scripts/gui-smoke.ps1 -Workspace . -OutDir target/devil-gui-smoke/phase6-dry-run -DryRun
sh scripts/gui-smoke.sh --workspace . --out-dir target/devil-gui-smoke/phase6-dry-run --dry-run
```

</verify>

### BLOCKED Conditions

- Safe dry-run behavior cannot be implemented for both scripts.
