# Plan 06-02: Windows Package And Launch Contract

## Objective

Produce a verified Windows release package path for the existing `devil-desktop` binary without adding unapproved installer dependencies.

<execution_contract>

- read-before-write: read `06-CONTEXT.md`, `06-01-RESULT.md`, `crates/devil-desktop/src/main.rs`, `crates/devil-desktop/src/workflow.rs`, `crates/devil-desktop/Cargo.toml`, `Cargo.toml`, `scripts/run-phase-gates.ps1`, and `plans/dependency-policy.md` before editing.
- evidence-before-action: confirm `devil-desktop` already has a binary entrypoint and no packaging helper module before creating package files.
- minimal diff: add packaging helpers, tests, one PowerShell script, one runbook, and result evidence only.
- verify-before-report: run packaging tests and the dry-run packaging script; record outcomes in `06-02-RESULT.md`.
- role: Windows packaging implementer with QA verification.
- task: create a deterministic release-package contract around `cargo build --release -p devil-desktop`.
- scope: desktop package helper, package tests, package script, package runbook, and plan result.
- allowed tools/actions: Rust helper/test edits, PowerShell script creation, `cargo test`, dry-run script execution, and `rg`.
- forbidden actions: adding production dependencies, changing workspace manifests, modifying app/UI authority, changing smoke/session behavior, or writing accepted final Phase 6 evidence.
- verification criteria: tests pass, dry-run script exits zero, package manifest includes binary path, version, workspace, launch examples, and no raw source payloads.
- final result format: write `.planning/phases/06-packaging-platform-integration-and-accessibility/06-02-RESULT.md` with changed files, command outcomes, generated dry-run path, and blockers.

</execution_contract>

<stop_gates>

- BLOCKED if a packaged executable cannot be represented without adding an installer dependency or changing `Cargo.toml`.
- BLOCKED if the package script would delete, overwrite, or recursively clean anything outside its explicit `-OutDir`.
- BLOCKED if package metadata would include source file contents, editor buffer text, terminal output payloads, provider payloads, or secrets.
- BLOCKED if `cargo test -p devil-desktop --test packaging` cannot run due to unrelated build failures that block packaging verification.

</stop_gates>

<recovery>

- If release build behavior is too slow for tests, keep tests in dry-run mode and document the actual release-build command in the runbook.
- If PowerShell path handling fails for spaces, fix quoting with `-LiteralPath` and typed `Join-Path`; do not shell out to `cmd` for file moves.
- If an installer is requested during execution, emit `BLOCKED` unless the same change includes dependency-policy and evidence-gate updates.
- On interruption, inspect `git diff`, compare modified files against `files_modified` and `files_forbidden`, rerun verification, and continue only from unverified work.

</recovery>

## Task 1: Add Desktop Package Manifest Helpers

### Read Targets

- `crates/devil-desktop/src/main.rs`
- `crates/devil-desktop/src/workflow.rs`
- `crates/devil-desktop/src/lib.rs`
- `crates/devil-desktop/Cargo.toml`
- `Cargo.toml`

### Write Targets

- `crates/devil-desktop/src/package.rs`
- `crates/devil-desktop/src/lib.rs`
- `crates/devil-desktop/tests/packaging.rs`

### Implementation Sequence

1. Create `crates/devil-desktop/src/package.rs` with `WindowsPackageManifest`, `WindowsPackageFile`, and `build_windows_package_manifest(workspace_root, package_root, binary_path)`. Include package name `Devil IDE`, binary name `devil-desktop.exe`, version from `env!("CARGO_PKG_VERSION")`, launch examples, and metadata-only notes.
   > verification: `rg -q "WindowsPackageManifest" crates/devil-desktop/src/package.rs`
2. Export the module from `crates/devil-desktop/src/lib.rs` with `pub mod package;`.
   > verification: `rg -q "pub mod package" crates/devil-desktop/src/lib.rs`
3. Add `crates/devil-desktop/tests/packaging.rs` covering manifest fields, path normalization, no raw-source markers, and dry-run package paths with spaces.
   > verification: `cargo test -p devil-desktop --test packaging -- --nocapture`

### Edge And Error Cases

- Manifest paths must be display metadata, not proof that files were copied.
- Reject empty package root or binary path.
- Keep helper free of filesystem mutation except path validation if needed.

<verify>

```powershell

rg -q "pub mod package" crates/devil-desktop/src/lib.rs
cargo test -p devil-desktop --test packaging -- --nocapture
```

</verify>

### BLOCKED Conditions

## Task 2: Add Dry-Run Capable Windows Packaging Script

### Read Targets

- `scripts/run-phase-gates.ps1`
- `crates/devil-desktop/src/package.rs`
- `plans/dependency-policy.md`

### Write Targets

- `scripts/package-windows.ps1`

### Implementation Sequence

1. Create `scripts/package-windows.ps1` with parameters `-Workspace`, `-OutDir`, and `[switch] -DryRun`. Resolve both paths, verify the output path is not the workspace root, and create/copy only inside `-OutDir` when not dry-run.
   > verification: `rg -q "param\\(" scripts/package-windows.ps1`
2. In non-dry-run mode, run `cargo build --release -p devil-desktop`, copy `target/release/devil-desktop.exe`, write `manifest.json`, and write a `RUNBOOK.txt` with launch examples. In dry-run mode, print the planned operations and write no package files.
   > verification: `rg -q "cargo build --release -p devil-desktop" scripts/package-windows.ps1`
3. Use PowerShell-native `Resolve-Path`, `Join-Path`, `New-Item`, and `Copy-Item -LiteralPath`; do not call `cmd`, `robocopy`, or destructive cleanup.
   > verification: `pwsh -NoProfile -File scripts/package-windows.ps1 -Workspace . -OutDir target/devil-package/phase6-dry-run -DryRun`

### Edge And Error Cases

- On non-Windows, the script may run dry-run only and must explain that `.exe` packaging is Windows-targeted.
- If `target/release/devil-desktop.exe` does not exist after build, fail with a clear error.
- Never delete a pre-existing output directory recursively.

<verify>

```powershell
rg -q "cargo build --release -p devil-desktop" scripts/package-windows.ps1
pwsh -NoProfile -File scripts/package-windows.ps1 -Workspace . -OutDir target/devil-package/phase6-dry-run -DryRun
```

</verify>

### BLOCKED Conditions

- Safe path validation cannot prove output files stay under `-OutDir`.
