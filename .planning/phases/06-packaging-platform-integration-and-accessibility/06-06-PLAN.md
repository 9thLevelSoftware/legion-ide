# Plan 06-06: Phase 6 Evidence Capture And Parity Plan

## Objective

Archive the evidence needed for GUI Phase 6 acceptance without modifying product code.

<execution_contract>

- read-before-write: read `06-CONTEXT.md`, `06-01-RESULT.md` through `06-05-RESULT.md`, all Phase 6 GUI evidence stubs/runbooks, `plans/evidence/gui-productization/phase-2-renderer-smoke.md`, `phase-3-session-and-large-file.md`, `phase-5-control-trust-safety.md`, and `AGENTS.md` before writing evidence.
- evidence-before-action: run or cite the exact commands that produce package, platform/accessibility, session/diagnostics, workflow, and performance evidence.
- minimal diff: write evidence Markdown and this result file only; do not modify product code or scripts in this wave.
- verify-before-report: run `cargo test -p devil-desktop --all-targets` plus targeted smoke/script commands that are available in the environment; record pass/fail/blocked outcomes honestly.
- role: evidence QA, performance verifier, and technical writer.
- task: capture package/platform/accessibility/session/diagnostics/workflow/performance evidence and document parity limitations.
- scope: Phase 6 GUI evidence files and plan result.
- allowed tools/actions: running tests/scripts/smoke commands, writing Markdown evidence, `rg`, and source/evidence reads.
- forbidden actions: editing product code, scripts, CI, manifests, or legacy collaboration evidence; marking acceptance in the main GUI Phase 6 evidence file.
- verification criteria: evidence files exist, contain command names and outcomes, distinguish passed/blocked/manual evidence, and do not overclaim unavailable OS observations.
- final result format: write `.planning/phases/06-packaging-platform-integration-and-accessibility/06-06-RESULT.md` with changed files, command outcomes, residual risks, and blockers.

</execution_contract>

<stop_gates>

- BLOCKED if package evidence cannot identify whether the package was dry-run, real release package, or blocked.
- BLOCKED if platform/accessibility evidence cannot include `accessibility_tree_smoke` and explicit status for IME, clipboard, focus, high-DPI, file-dialog, menu, shortcut, and theme checks.
- BLOCKED if session/diagnostics evidence includes raw source, terminal payloads, provider payloads, or secrets.
- BLOCKED if any evidence file claims final acceptance before Plan 06-07.

</stop_gates>

<recovery>

- If native window smoke is blocked locally, write the blocked smoke report and record the exact blocker; do not convert blocked to passed.
- If `cargo test -p devil-desktop --all-targets` fails, classify whether the failure is Phase 6-related. Fix only if within this plan's evidence scope; otherwise stop with `BLOCKED`.
- If scripts are unavailable on the host, record exact command and environment limitation.
- On interruption, inspect `git diff`, compare modified files against `files_modified` and `files_forbidden`, rerun verification, and continue only from unverified work.

</recovery>

<verification>

- [ ] Package evidence exists and cites package command output.
- [ ] Platform/accessibility evidence includes deterministic accessibility tree status and OS-observed/adaptor-path labels.
- [ ] Session/diagnostics evidence proves crash-safe metadata-only behavior.
- [ ] Workflow smoke evidence covers package/install, launch, open, edit/save, terminal, language, proposal, diagnostics, and quit.
- [ ] Performance/reliability evidence includes p50/p95/frame variance and residual risks.
- [ ] No product code, scripts, CI, manifests, or legacy evidence are modified.

</verification>

<success_criteria>

- GUI Phase 6 acceptance has an evidence package to evaluate.
- Residual risks are explicit and not hidden in acceptance language.
- Plan 06-07 can make an evidence-gated go/no-go decision.

</success_criteria>
