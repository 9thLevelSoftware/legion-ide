# Plan 06-04: Crash-Safe Session And Diagnostics Export

## Objective

Make crash-safe session restore and metadata-only diagnostics export available through the desktop adapter.

<execution_contract>

- read-before-write: read `06-CONTEXT.md`, `06-03-RESULT.md`, `crates/devil-desktop/src/session.rs`, `crates/devil-desktop/src/workflow.rs`, `crates/devil-desktop/src/smoke.rs`, `crates/devil-desktop/tests/session_restore.rs`, and `crates/devil-cli/src/main.rs` before editing.
- evidence-before-action: confirm current session writes use direct `fs::write` and diagnostics export does not already exist.
- minimal diff: keep changes inside `devil-desktop` session/diagnostics/workflow/tests and this result file.
- verify-before-report: run session restore and diagnostics export tests and record outcomes in `06-04-RESULT.md`.
- role: reliability implementer with QA verification.
- task: make session persistence write-then-validate-then-rename and add redacted diagnostics export based on projections/status metadata.
- scope: desktop session persistence, diagnostics export module, launch arg parsing, tests, and result.
- allowed tools/actions: Rust code/test edits under `devil-desktop`, `cargo test`, and `rg`.
- forbidden actions: persisting source text, changing app/session DTO ownership, adding storage authority to desktop, changing protocol DTOs, or adding dependencies.
- verification criteria: corrupt/temp session scenarios preserve last good session; diagnostics export rejects raw markers and contains package/platform/session/smoke metadata only.
- final result format: write `.planning/phases/06-packaging-platform-integration-and-accessibility/06-04-RESULT.md` with changed files, command outcomes, and blockers.

</execution_contract>

- BLOCKED if crash-safe replacement cannot be implemented without destructive cleanup or platform-specific unsafe assumptions.
- BLOCKED if diagnostics export requires reading editor source text, terminal output payloads, provider payloads, secrets, or raw storage records.

</stop_gates>

- If launch argument parsing conflicts with smoke args, keep diagnostics export optional and covered by explicit tests.

</recovery>

## Task 1: Make Session Saves Crash-Safe

### Read Targets

- `crates/devil-desktop/src/session.rs`
- `crates/devil-desktop/tests/session_restore.rs`

### Write Targets

- `crates/devil-desktop/src/session.rs`
- `crates/devil-desktop/tests/session_restore.rs`

### Implementation Sequence

1. Add `DesktopSessionStore::save_crash_safe(path, record)` and route `save` through it unless keeping `save` as a wrapper is cleaner. The implementation must serialize, reject raw markers, write a sibling temp file, read/validate it back, then rename to the final path.
   > verification: `rg -q "save_crash_safe" crates/devil-desktop/src/session.rs`
2. Use a deterministic temp file name based on the final file name plus process id, not a random global path. Temp writes must stay in the same directory as the final session file.
   > verification: `rg -q "with_extension" crates/devil-desktop/src/session.rs`
3. Add tests for valid save/restore, corrupt existing final file, temp file left behind, raw marker rejection, and path parent creation.
   > verification: `cargo test -p devil-desktop --test session_restore -- --nocapture`

### Edge And Error Cases

- Missing session file remains a no-op on load.
- Corrupt final session still returns a typed JSON error.
- A failed temp write must leave the previous final file untouched.

<verify>

```powershell
rg -q "save_crash_safe" crates/devil-desktop/src/session.rs
rg -q "with_extension" crates/devil-desktop/src/session.rs
cargo test -p devil-desktop --test session_restore -- --nocapture
```

</verify>

### BLOCKED Conditions

- Safe same-directory temp replacement cannot be implemented without broad platform changes.

## Task 2: Add Metadata-Only Diagnostics Export

### Read Targets

- `crates/devil-desktop/src/workflow.rs`
- `crates/devil-desktop/src/smoke.rs`
- `crates/devil-desktop/src/platform.rs`
- `crates/devil-desktop/src/session.rs`

### Write Targets

- `crates/devil-desktop/src/diagnostics.rs`
- `crates/devil-desktop/src/lib.rs`
- `crates/devil-desktop/tests/diagnostics_export.rs`

### Implementation Sequence

1. Create `DesktopDiagnosticsExport` with fields for app version, workspace path label, open tab count, dirty count, status counts, platform snapshot labels, smoke summary labels when available, session state path label, and generated timestamp label. Do not include buffer text, terminal output payloads, provider payloads, or storage records.
   > verification: `rg -q "DesktopDiagnosticsExport" crates/devil-desktop/src/diagnostics.rs`
2. Add `to_markdown()` and `write_diagnostics_export(path, export)` with raw-marker rejection using the same marker family as session persistence plus `terminal_output`, `provider_payload`, `raw_prompt`, and `secret`.
   > verification: `rg -q "write_diagnostics_export" crates/devil-desktop/src/diagnostics.rs`
3. Export `pub mod diagnostics;` and add tests for markdown fields, raw marker rejection, and no source text from a dirty buffer.
   > verification: `cargo test -p devil-desktop --test diagnostics_export -- --nocapture`

### Edge And Error Cases

- Empty workspace/projection should still export shell metadata and a warning status.
- Reject suspicious marker strings even if they appear in labels.
- Diagnostics export errors must not mark desktop workflow as successful package acceptance.

<verify>

```powershell

rg -q "write_diagnostics_export" crates/devil-desktop/src/diagnostics.rs
cargo test -p devil-desktop --test diagnostics_export -- --nocapture
```

</verify>

### BLOCKED Conditions

- Diagnostics export cannot be produced without raw source or app internals.
