# Plan 06-03: Native Platform Integration Smoke Model

## Objective

Make native platform integration smoke evidence explicit and deterministic for menus, file dialogs, clipboard, shortcuts, theme, focus traversal, IME, high-DPI, and accessibility tree behavior without moving authority into the renderer.

<execution_contract>

- read-before-write: read `06-CONTEXT.md`, `06-02-RESULT.md`, `crates/devil-desktop/src/workflow.rs`, `crates/devil-desktop/src/smoke.rs`, `crates/devil-desktop/src/bridge.rs`, `crates/devil-desktop/src/view.rs`, `crates/devil-desktop/tests/platform_smoke.rs`, and `plans/adrs/ADR-0030-desktop-adapter-boundary.md` before editing.
- evidence-before-action: identify current `not observed` smoke fields and bridge adapter-path checks before changing report semantics.
- minimal diff: keep changes in `devil-desktop` platform/smoke/workflow tests; do not alter app, UI, protocol, or manifests.
- verify-before-report: run platform integration and platform smoke tests and record outcomes in `06-03-RESULT.md`.
- role: desktop platform integration implementer with UI/accessibility review.
- task: add deterministic platform/accessibility smoke model and wire it into smoke evidence without overclaiming OS observations.
- scope: desktop adapter platform snapshot, smoke report population, native window setup labels, tests, and result.
- allowed tools/actions: Rust code/test edits under `devil-desktop`, `cargo test`, and `rg`.
- forbidden actions: adding new dependencies, taking editor/workspace/proposal/terminal/provider authority, editing protocol DTOs, or converting unavailable OS facts into passing claims.
- verification criteria: tests prove fields exist, labels distinguish `os-observed`, `adapter-path`, `accessibility-tree model`, and `not observed`, and no raw input payloads are recorded.
- final result format: write `.planning/phases/06-packaging-platform-integration-and-accessibility/06-03-RESULT.md` with changed files, command outcomes, observed limitations, and blockers.

</execution_contract>

<stop_gates>

- BLOCKED if accessibility smoke cannot provide at least a deterministic adapter-local accessibility tree model for shell regions, editor, explorer, status, trust, terminal, and assistant surfaces.
- BLOCKED if platform smoke can only pass by claiming `os-observed` values that are not actually observed by egui/eframe input.
- BLOCKED if native menu/file-dialog/clipboard/IME actions would mutate app/editor/workspace state outside `DesktopCommandBridge` and `AppComposition`.
- BLOCKED if testing requires modifying `devil-ui`, `devil-app`, or `devil-protocol` in this wave.

</stop_gates>

<recovery>

- If OS-level accessibility cannot be observed in headless tests, implement deterministic accessibility tree projection rows and label the field `accessibility-tree model passed`.
- If a native menu bar is not portable in egui without more dependencies, expose a desktop command menu model and keyboard shortcut table, then document OS menu parity as a residual risk for Plan 06-06.
- If high-DPI is unavailable in CI, keep `high_dpi_smoke` as `not observed` in headless mode and require manual/Windows smoke evidence before acceptance.
- On interruption, inspect `git diff`, compare modified files against `files_modified` and `files_forbidden`, rerun verification, and continue only from unverified work.

</recovery>

<verification>

- [ ] `DesktopPlatformIntegrationSnapshot` exists and is metadata-only.

- [ ] Platform tests cover labels and no raw payload storage.
- [ ] Existing editor input, clipboard paste, IME commit, and file dialog bridge tests still pass.
- [ ] No app/UI/protocol/manifest files are modified.

</verification>

<success_criteria>

- Phase 6 platform behavior can be evidenced without ambiguity between adapter-path and OS-observed facts.
- Accessibility tree smoke has a deterministic model and test coverage.
- Native integration remains an adapter concern, not product-state authority.

</success_criteria>

## Task 1: Add Platform Integration Snapshot Model

### Read Targets

- `crates/devil-desktop/src/bridge.rs`
- `crates/devil-desktop/src/view.rs`
- `crates/devil-desktop/src/workflow.rs`
- `plans/adrs/ADR-0030-desktop-adapter-boundary.md`

### Write Targets

- `crates/devil-desktop/src/platform.rs`
- `crates/devil-desktop/src/lib.rs`
- `crates/devil-desktop/tests/platform_integration.rs`

### Implementation Sequence

1. Create `DesktopPlatformIntegrationSnapshot` with fields for menu model, keyboard shortcuts, file dialog path, clipboard adapter path, IME commit path, theme label, focus traversal labels, high-DPI observation label, and accessibility tree summary. Values must be labels/counts only.
   > verification: `rg -q "DesktopPlatformIntegrationSnapshot" crates/devil-desktop/src/platform.rs`
2. Add `build_platform_integration_snapshot(snapshot: &ShellProjectionSnapshot) -> DesktopPlatformIntegrationSnapshot` that derives rows from the current projection and bridge capabilities without storing payload text.
   > verification: `rg -q "build_platform_integration_snapshot" crates/devil-desktop/src/platform.rs`
3. Export `pub mod platform;` and add tests proving expected shell regions, shortcut labels, accessibility nodes, and absence of raw markers such as `SECRET_DIRTY_BODY`, `provider_payload`, and `terminal_output`.
   > verification: `cargo test -p devil-desktop --test platform_integration -- --nocapture`

### Edge And Error Cases

- If a projection is empty, still produce shell-level accessibility nodes for window, explorer, editor, status, and trust surfaces with empty-state labels.
- Do not store actual clipboard/IME text; store capability labels only.
- Do not depend on OS APIs in unit tests.

<verify>

```powershell
rg -q "DesktopPlatformIntegrationSnapshot" crates/devil-desktop/src/platform.rs
rg -q "build_platform_integration_snapshot" crates/devil-desktop/src/platform.rs
cargo test -p devil-desktop --test platform_integration -- --nocapture
```

</verify>

### BLOCKED Conditions

- A metadata-only platform snapshot cannot be derived from the existing projections.
