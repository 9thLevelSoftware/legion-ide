# Plan 03-05: Session Restore And Large-File Guardrails

## Objective

Persist and restore daily editing session metadata, then prove desktop rendering/search keeps large-file degraded mode bounded.

<execution_contract>

- read-before-write: read Plans 03-01 through 03-04 results, `WorkspaceSessionRecord` DTOs, desktop launch/workflow/smoke code, editor degraded-mode tests, and Phase 2 smoke evidence.
- evidence-before-action: use `WorkspaceSessionRecord` metadata and existing app restore/capture helpers; do not persist source bodies.
- minimal diff: add desktop session helper, tests, and focused evidence only.
- verify-before-report: run session/large-file tests and the editor performance-suite listing.
- role: senior desktop implementer plus performance benchmarker.
- task: make repeated sessions recover workspace/tabs/focus/layout/explorer metadata and preserve large-file safety.
- scope: desktop session store, launch config, restore/apply workflow, metadata-only JSON, focused evidence, large-file GUI/search guardrails.
- allowed tools/actions: add `serde_json` to `devil-desktop` if needed from existing workspace dependencies, edit desktop files, write tests/evidence.
- forbidden actions: storing raw buffer text, adding `devil-storage` dependency to `devil-desktop`, modifying editor/text degraded-mode internals, or enabling ignored 100MB workload as a passing claim.
- verification criteria: restored session recovers metadata; dirty text is not persisted; degraded view/search uses bounded viewport/chunk paths.
- final result format: write `.planning/phases/03-daily-editing-mvp/03-05-RESULT.md`.

</execution_contract>

</stop_gates>

</recovery>

## Task 1: Add Metadata-Only Session Store

### Read Targets

- `crates/devil-protocol/src/lib.rs`
- `crates/devil-desktop/src/workflow.rs`
- `crates/devil-desktop/Cargo.toml`

### Write Targets

- `crates/devil-desktop/Cargo.toml`
- `crates/devil-desktop/src/lib.rs`
- `crates/devil-desktop/src/session.rs`
- `crates/devil-desktop/src/workflow.rs`

### Implementation Sequence

1. Add `serde_json = { workspace = true }` to `crates/devil-desktop/Cargo.toml` only if `session.rs` needs JSON serialization. Do not add new external crates.
   > verification: `rg -q "serde_json" crates/devil-desktop/Cargo.toml`
2. Add `pub mod session` and implement `DesktopSessionStore` with explicit `load(path) -> Result<Option<WorkspaceSessionRecord>>` and `save(path, &WorkspaceSessionRecord) -> Result<()>` APIs.
   > verification: `rg -q "pub mod session" crates/devil-desktop/src/lib.rs`
3. Reject session records with `schema_version == 0`, empty `session_id`, or suspicious raw-source markers such as `"small_buffer_preview"`, `"source_body"`, or dirty edited body fixture strings in serialized output.
   > verification: `rg -q "schema_version" crates/devil-desktop/src/session.rs`
4. Extend `DesktopLaunchConfig` with `--session-state <path>` and optional restore behavior. Keep default deterministic and document if no session path is supplied.
   > verification: `rg -q "session-state" crates/devil-desktop/src/workflow.rs`

### Edge And Error Cases

- Corrupt JSON returns a typed error and does not open arbitrary paths.
- Missing session file is a no-op.
- Session restore skips missing files and reports skipped tabs.

<verify>

```powershell
cargo check -p devil-desktop --all-targets
```

</verify>

### BLOCKED Conditions

- Session persistence requires raw source content or storage internals.

- Dirty indicators are displayed as metadata, not replayed as unsaved text.

### Implementation Sequence

   > verification: `rg -q "large_file_degraded_status" crates/devil-desktop/src/smoke.rs`

   > verification: `rg -q "100MB performance workload remains" plans/evidence/gui-productization/phase-3-session-and-large-file.md`

- Do not unignore performance workloads to force a green benchmark.
- Do not claim streaming-mode completion beyond what tests prove.
