# Plan 03-06: Phase 3 Evidence And Acceptance Gate

## Objective

Prove Phase 3 meets the Daily Editing MVP success criteria from live commands and artifact evidence, then update planning state only if the evidence supports acceptance.

<execution_contract>

- read-before-write: read all Phase 3 `*-RESULT.md` files, current source diffs, `03-CONTEXT.md`, `.planning/ROADMAP.md`, `.planning/STATE.md`, Phase 2 review/evidence, and all command outputs produced by this plan.
- evidence-before-action: do not mark Phase 3 accepted until targeted daily-editing tests, desktop all-target tests, workspace gates, and evidence checks are run and recorded.
- minimal diff: documentation/state updates only; no crate source changes in this final gate.
- verify-before-report: run all frontmatter verification commands or record exact blockers.
- role: QA verifier and technical writer.
- task: archive acceptance evidence and reconcile Phase 3 planning state.
- scope: final evidence markdown, wave checklist, result artifact, roadmap progress, and state next action.
- allowed tools/actions: read result files/source, run cargo gates, write evidence/state docs.
- forbidden actions: modifying source code, weakening gates, marking failed/missing evidence as accepted, or hiding residual risk.
- verification criteria: success criteria table has `met`, `blocked`, or `not met` for every roadmap criterion with source/test evidence.
- final result format: write `.planning/phases/03-daily-editing-mvp/03-06-RESULT.md` and final Phase 3 evidence.

</execution_contract>

<stop_gates>

- BLOCKED if any prior Phase 3 plan result is missing.
- BLOCKED if a targeted test or gate fails and the root cause is in Phase 3 scope.
- BLOCKED if final evidence cannot truthfully mark every Phase 3 success criterion met.
- BLOCKED if source diff shows `devil-ui` owns editor/workspace state or renderer dependencies.

</stop_gates>

<recovery>

- If interrupted, read `WAVE-CHECKLIST.md`, inspect `git diff`, compare modified files against `files_modified`/`files_forbidden`, rerun any verification command lacking recorded output, and continue only from unverified evidence sections.
- If a broad workspace command fails from an unrelated pre-existing issue, record exact output, prove targeted Phase 3 commands pass, and do not mark the broad gate passed.

</recovery>

- [ ] All prior Phase 3 plan results exist and summarize their verification.

- [ ] Evidence document includes artifact inventory, command table, success-criteria decision table, boundary proof, search proof, session proof, conflict proof, large-file proof, and residual risks.
- [ ] State and roadmap are updated only after evidence supports acceptance.

</verification>
