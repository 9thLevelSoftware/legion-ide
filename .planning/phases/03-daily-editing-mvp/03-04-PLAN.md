# Plan 03-04: Save-All Conflict And Dirty-Close Hardening

## Objective

Harden the highest-risk daily editing flows: save-all, external overwrite conflicts, visible save rejection state, and dirty-close prompts.

<execution_contract>

- read-before-write: read Plans 03-01 through 03-03 results, `crates/devil-app/src/lib.rs`, `crates/devil-desktop/src/workflow.rs`, existing save conflict tests, and Phase 2 evidence.
- evidence-before-action: preserve Phase 2 save rejection semantics and reuse `SaveWorkflowService`; prove dirty text stays dirty on rejected save and cancelled dirty close.
- minimal diff: harden app/desktop transitions and tests only.
- verify-before-report: run the listed save/conflict tests and record command output.
- role: senior reliability implementer plus QA verification specialist.
- task: make save-all/conflict/dirty-close behavior safe enough for repeated local editing.
- scope: aggregate save-all outcome, conflict projection/status, close-dirty prompt, rejection-preservation tests.
- allowed tools/actions: targeted source edits, temp-workspace tests, cargo tests.
- forbidden actions: direct workspace writes, clearing dirty state on rejection, deleting dirty buffers without explicit verified discard, or suppressing save errors.
- verification criteria: rejected saves preserve dirty text, conflicts are visible, clean buffers save, dirty-close prompts block close until resolved.
- final result format: write `.planning/phases/03-daily-editing-mvp/03-04-RESULT.md`.

</execution_contract>

<stop_gates>

- BLOCKED if save-all cannot target each buffer with correct file metadata and non-zero correlation/causality through the existing save workflow.
- BLOCKED if conflict projection cannot distinguish stale/conflict/rejected from saved.
- BLOCKED if close-dirty behavior would require losing editor text.

</stop_gates>

<recovery>

- If interrupted, inspect `git diff`, rerun the targeted save-all and conflict tests, and compare behavior against Phase 2 rejection preservation.
- If aggregate save-all state is unclear, make the app result more explicit rather than encoding success-shaped defaults.

</recovery>

<verification>

- [ ] Save-all over multiple clean buffers saves all files.
- [ ] Save-all with one external overwrite saves independent clean buffers and leaves conflicted dirty buffer unchanged.
- [ ] Dirty-close prompt cancellation preserves dirty text and tab.
- [ ] Phase 2 external-overwrite regression still passes.

</verification>

<success_criteria>

- Users can save all open buffers from the GUI and see per-buffer saved/rejected results.
- External overwrite conflicts remain visible and dirty text is preserved.
- Close-dirty prompt behavior is deterministic and tested.

</success_criteria>

<output>

Write `.planning/phases/03-daily-editing-mvp/03-04-RESULT.md`.

</output>

## Task 1: Harden App Save-All Outcomes

### Read Targets

- `crates/devil-app/src/lib.rs`
- `crates/devil-app/tests/daily_editing_contracts.rs`
- `crates/devil-app/tests/workspace_vfs_integration.rs`

### Write Targets

- `crates/devil-app/src/lib.rs`
- `crates/devil-app/tests/daily_editing_contracts.rs`

### Implementation Sequence

1. Define explicit app save-all result records with buffer id, file id/path, saved/rejected status, proposal response metadata for rejections, and final dirty state.
   > verification: `rg -q "SaveAll" crates/devil-app/src/lib.rs`
2. Iterate open buffers sequentially. Temporarily bind the target buffer's metadata for the existing `SaveWorkflowService` call or extract a shared buffer-specific save helper; do not call workspace save APIs directly.
   > verification: `rg -q "bind_saved_buffer" crates/devil-app/src/lib.rs`
3. Preserve rejected dirty buffers by acknowledging failures with `acknowledgement_for_response` and never marking rejected buffers clean.
   > verification: `rg -q "Rejected" crates/devil-app/src/lib.rs`
4. Add app tests for all-clean save-all, mixed saved/rejected save-all, missing metadata error, and aggregate result ordering.
   > verification: `cargo test -p devil-app daily_editing_save_all -- --nocapture`

### Edge And Error Cases

- Continue after a rejected buffer, but report aggregate status as partial/rejected.
- Missing metadata for any buffer returns an error for that buffer and does not save it.
- No open buffers returns a typed no-op outcome.

<verify>

```powershell
cargo test -p devil-app daily_editing_save_all -- --nocapture
```

</verify>

### BLOCKED Conditions

- Existing app state lacks enough metadata to build save preconditions for non-active buffers.

### Read Targets

- `crates/devil-desktop/src/view.rs`
- `crates/devil-desktop/src/workflow.rs`
- `crates/devil-desktop/tests/save_all_conflict.rs`

### Implementation Sequence

1. Route desktop save-all action to the Plan 03-01/03-04 app intent and map aggregate results into `DesktopWorkflowOutcome`.
   > verification: `rg -q "SaveAll" crates/devil-desktop/src/workflow.rs`
2. Extend view-model/status rows so stale/conflict/denied save rejections display as warning/error rows with file/tab identity and do not look like success.
   > verification: `rg -q "conflict" crates/devil-desktop/src/view.rs`
3. Implement dirty-close prompt states for cancel, save, discard only if the app provides a verified discard path, and close-clean. Default dirty close must prompt and preserve.
   > verification: `rg -q "close_dirty" crates/devil-desktop/src/workflow.rs`
4. Add desktop tests for mixed save-all conflict, prompt cancel preservation, prompt save route, and clean close removal.
   > verification: `cargo test -p devil-desktop save_all_conflict -- --nocapture`

### Edge And Error Cases

- If discard is not implemented by app contract, do not show a functional discard action; show cancel/save only.
- Rejected save must leave the active tab dirty and status warning visible.
- Prompt-active keyboard/text events must not edit buffer text.
