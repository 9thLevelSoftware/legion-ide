# Plan 03-02: Desktop Tabs Explorer And Viewport Controls

## Objective

Render and route the daily editing controls introduced by Plan 03-01 inside `devil-desktop` while keeping the adapter as a projection consumer and intent emitter.

<execution_contract>

- read-before-write: read Plan 03-01 result, `crates/devil-desktop/src/bridge.rs`, `view.rs`, `workflow.rs`, existing desktop tests, and `crates/devil-ui/src/ui.rs`.
- evidence-before-action: use the Plan 03-01 `CommandDispatchIntent` and projection fields exactly; do not add app logic to `view.rs` or `bridge.rs`.
- minimal diff: change only desktop adapter source/tests.
- verify-before-report: run all frontmatter commands and record results.
- role: UI implementation specialist with senior Rust support.
- task: make tab, explorer, cursor, selection, scroll, save-all, and dirty-close interactions usable in the desktop shell.
- scope: adapter-local actions, rendering, keyboard/mouse routing, prompt state, and desktop tests.
- allowed tools/actions: edit desktop adapter files, add tests, run desktop cargo tests/checks.
- forbidden actions: modifying app/UI contracts in this plan, importing editor/project/storage internals into desktop view/bridge, or direct disk writes.
- verification criteria: every desktop action translates to an app intent/request or typed no-op/error; no dirty close loses text silently.
- final result format: write `.planning/phases/03-daily-editing-mvp/03-02-RESULT.md`.

</execution_contract>

<stop_gates>

- BLOCKED if Plan 03-01 command/projection names are missing or changed without a compatible adapter path.
- BLOCKED if an interaction needs app state not exposed in `ShellProjectionSnapshot` or `DesktopRuntime` public methods.
- BLOCKED if implementing cursor/selection requires direct editor access from desktop.

</stop_gates>

<recovery>

- On interruption, inspect `git diff`, verify no app/UI files were modified, rerun the desktop tests that cover completed controls, and continue from unverified controls.
- If egui APIs cannot express a control cleanly, keep the view-model/test layer first and add minimal rendering only after the behavior is proven.

</recovery>

<verification>

- [ ] Bridge translation tests cover all new actions.
- [ ] View-model tests cover tab rows, explorer expansion/selection, viewport cursor/selection/scroll flags, dirty prompts, empty states, and degraded state.
- [ ] Workflow tests prove runtime dispatch uses app authority.

</verification>

<success_criteria>

- The desktop shell can switch tabs, request clean/dirty close, save all, select/reveal explorer files, update cursor/selection, and scroll the viewport.
- Close-dirty prompts are visible in projection/view model state and route to app-owned outcomes.
- Existing Phase 2 open/edit/save/rejection/quit tests still pass.

</success_criteria>

Write `.planning/phases/03-daily-editing-mvp/03-02-RESULT.md` with implementation and verification evidence.

## Task 1: Extend Desktop Action Bridge

### Read Targets

- `crates/devil-desktop/src/bridge.rs`
- `crates/devil-desktop/tests/intent_bridge.rs`
- `crates/devil-ui/src/ui.rs`

### Write Targets

- `crates/devil-desktop/src/bridge.rs`
- `crates/devil-desktop/tests/intent_bridge.rs`

### Implementation Sequence

1. Add `DesktopAction` variants for `SwitchTab`, `CloseTab`, `SaveAll`, `SetCursor`, `SetSelection`, `SetViewportScroll`, `ToggleExplorerPath`, and `SelectExplorerFile`.
   > verification: `rg -q "SwitchTab" crates/devil-desktop/src/bridge.rs`
2. Map buffer/file actions into Plan 03-01 `CommandDispatchIntent` variants. Explorer expansion is adapter-local unless the app contract exposes it; file selection/reveal must use app-owned file identifiers.
   > verification: `rg -q "SetViewportScroll" crates/devil-desktop/src/bridge.rs`
3. Return typed `DesktopBridgeError` values for missing active buffer, unknown tab, unknown explorer file, empty path, and unsupported actions.
   > verification: `rg -q "UnknownTab" crates/devil-desktop/src/bridge.rs`
4. Extend `intent_bridge` tests to assert all new actions map correctly and no bridge code imports `AppComposition`, `EditorEngine`, or `WorkspaceActor`.
   > verification: `cargo test -p devil-desktop intent_bridge -- --nocapture`

### Edge And Error Cases

- Save-all does not require an active buffer if the app projection says tabs exist.
- Cursor/selection/scroll actions require an active or target buffer and return typed errors otherwise.
- Explorer expand/collapse should not mutate app/workspace state.

<verify>

```powershell
cargo test -p devil-desktop intent_bridge -- --nocapture
```

</verify>

### BLOCKED Conditions

- The app/UI contract cannot represent a required action without modifying Plan 03-01 outputs.

## Task 2: Render Daily Editing View Model

### Read Targets

- `crates/devil-desktop/src/view.rs`
- `crates/devil-desktop/tests/projection_rendering.rs`
- `crates/devil-ui/src/ui.rs`

### Write Targets

- `crates/devil-desktop/src/view.rs`
- `crates/devil-desktop/tests/projection_rendering.rs`

### Implementation Sequence

1. Extend `DesktopProjectionViewModel` with `tab_rows`, `editor_status_rows`, `close_prompt_rows`, `explorer_state_rows`, and viewport metadata rows.
   > verification: `rg -q "tab_rows" crates/devil-desktop/src/view.rs`
2. Render tab strip, close indicators, save-all state, explorer rows with expansion/selection markers, cursor/selection summary, scroll position, and degraded-mode large-file status.
   > verification: `rg -q "close_prompt_rows" crates/devil-desktop/src/view.rs`
3. Keep editor content rendering bounded: prefer `ViewportProjection` line slices and use `small_buffer_text()` only when `active.degraded == false`.
   > verification: `rg -q "DegradedLargeFile" crates/devil-desktop/src/view.rs`
4. Extend projection rendering tests for clean/dirty tabs, no active tab, close prompt, selected explorer file, expanded/collapsed markers, cursor/selection rows, viewport scroll rows, and degraded large-file rows.
   > verification: `cargo test -p devil-desktop projection_rendering -- --nocapture`

### Edge And Error Cases

- No active buffer shows a stable empty editor state.
- Dirty tabs must include a dirty indicator without reading raw editor text.
- Long paths should be displayed from canonical path metadata only.

<verify>

```powershell

```

</verify>

### BLOCKED Conditions

## Task 3: Wire Workflow And Keyboard Controls

### Read Targets

- `crates/devil-desktop/src/workflow.rs`
- `crates/devil-desktop/tests/desktop_workflow.rs`
- `crates/devil-desktop/tests/daily_editing_controls.rs`

### Write Targets

- `crates/devil-desktop/src/workflow.rs`
- `crates/devil-desktop/tests/daily_editing_controls.rs`

### Implementation Sequence

1. Add keyboard routing for save-all, tab switching, tab close, cursor movement, selection update, viewport scroll, explorer refresh/reveal, and close-dirty prompt actions. Keep text input disabled while any modal prompt is active.
   > verification: `rg -q "SaveAll" crates/devil-desktop/src/workflow.rs`
2. Maintain adapter-local explorer expansion and prompt state in `DesktopRuntime`; app-owned state changes still go through `handle_action` -> bridge -> `AppComposition::dispatch_ui_intent`.
   > verification: `rg -q "explorer_expansion" crates/devil-desktop/src/workflow.rs`
3. Add `daily_editing_controls` tests for tab switch/close, save-all routing, prompt-active text suppression, cursor/selection/scroll dispatch, explorer select/reveal, and existing edit/save behavior.
   > verification: `cargo test -p devil-desktop daily_editing_controls -- --nocapture`

### Edge And Error Cases

- Prompt-active text and paste events must not route into the editor.
- Close-dirty prompt cancellation must leave the tab and buffer open.
- Keyboard shortcuts must not panic when no active buffer exists.

<verify>

```powershell
cargo test -p devil-desktop daily_editing_controls -- --nocapture
cargo test -p devil-desktop --all-targets
```

</verify>

### BLOCKED Conditions

- Prompt handling cannot distinguish editor text input from prompt input.
