## Task 1: Extend Projection-Only UI Contracts

### Read Targets

- `crates/devil-ui/src/ui.rs`
- `crates/devil-protocol/src/lib.rs`
- `crates/devil-protocol/tests/dto_contracts.rs`

### Write Targets

- `crates/devil-ui/src/ui.rs`

### Implementation Sequence

1. Add projection structs named `EditorTabProjection`, `EditorTabsProjection`, `CloseDirtyPromptProjection`, and `DailyEditingProjection`. Fields must use protocol DTOs where available: `BufferId`, `FileId`, `CanonicalPath`, `ViewportScroll`, `ProtocolTextRange`, and `WorkspaceSessionRecord` references only as metadata. Do not include raw text fields.
   > verification: `rg -q "pub struct EditorTabsProjection" crates/devil-ui/src/ui.rs`
2. Add a `daily_editing_projection: DailyEditingProjection` field to `ShellProjectionSnapshot` and wire it through `Shell`, `Shell::empty`, and `replace_projection_snapshot`.
   > verification: `rg -q "daily_editing_projection" crates/devil-ui/src/ui.rs`
3. Add `CommandDispatchIntent` variants `SwitchTab { buffer_id }`, `CloseTab { buffer_id }`, `SaveAll`, `SetCursor { buffer_id, cursor }`, `SetSelection { buffer_id, range }`, and `SetViewportScroll { buffer_id, scroll }`.
   > verification: `rg -q "SaveAll" crates/devil-ui/src/ui.rs`

### Edge And Error Cases

- Default projections must be empty and metadata-only.
- `devil-ui` must not import `devil_app`, `devil_editor`, `devil_project`, `devil_storage`, or `devil_desktop`.
- If a projection needs future layout details, add optional metadata fields rather than renderer objects.

## Task 2: Add App-Owned Daily Editing State And Routing

### Read Targets

- `crates/devil-app/src/lib.rs`
- `crates/devil-editor/src/lib.rs`
- `crates/devil-project/src/lib.rs`
- `crates/devil-protocol/src/lib.rs`

### Write Targets

- `crates/devil-app/src/lib.rs`

### Implementation Sequence

1. Extend `ActiveDocumentController` with explicit tab order and active tab metadata derived from opened buffers. Reuse existing `buffer_file_metadata`; add only the minimum tab metadata needed for projection and restore.
   > verification: `rg -q "open_tabs" crates/devil-app/src/lib.rs`
2. Extend `AppCommandRequest`, `CommandDispatcher::route_intent`, and `CommandExecutionService::execute` for `SwitchTab`, `CloseTab`, `SaveAll`, `SetCursor`, `SetSelection`, and `SetViewportScroll`.
   > verification: `rg -q "SetViewportScroll" crates/devil-app/src/lib.rs`
3. Use `EditorEngine::set_cursors` and `EditorEngine::set_selections` for cursor/selection updates. Store viewport scroll in app state and pass it into `EditorViewportRequest` in `ProjectionBuilder::active_buffer_projection`.
   > verification: `rg -q "set_cursors" crates/devil-app/src/lib.rs`
4. Implement save-all sequentially over open buffers using the existing `SaveWorkflowService` and `ActiveDocumentController::bind_saved_buffer`; rejected saves remain `Ok(AppSaveOutcome::Rejected(_))`-style outcomes and preserve dirty buffers.
   > verification: `rg -q "save_all" crates/devil-app/src/lib.rs`
5. Implement close-tab as a request/outcome that refuses dirty buffers unless the explicit close policy says to discard. The default route used by desktop must prompt/reject rather than close dirty.
   > verification: `rg -q "CloseDirty" crates/devil-app/src/lib.rs`
6. Add app helper methods to capture and restore `WorkspaceSessionRecord` metadata from open tabs, active tab, explorer expansion, panel state, and dirty indicators. Do not include buffer text.
   > verification: `rg -q "WorkspaceSessionRecord" crates/devil-app/src/lib.rs`

### Edge And Error Cases

- Switching to a closed/unknown buffer returns a typed app error and does not change active state.
- Save-all continues after a rejected buffer, records per-buffer status, and reports aggregate rejection without clearing dirty text.
- Close-clean calls `EditorEngine::close_buffer`; close-dirty returns a prompt/rejected outcome by default.
- Restoring a session with missing files records skipped tabs and continues opening valid tabs.
- Session capture must store dirty indicators only, not dirty text.

<verify>

```powershell

cargo check -p devil-app --all-targets
```

</verify>

### BLOCKED Conditions

- App state cannot distinguish clean close from dirty close without modifying `devil-editor`.
- Save-all requires direct workspace writes outside the existing save workflow.
