# Plan 08-04: Remote Workspace Manager And Remote Status GUI Workflow

## Objective

Expose remote workspace connection, remote session status, remote terminal/LSP descriptors, reconnect/offline indicators, and remote proposal review in the GUI without giving UI or desktop ownership of remote runtime, local disk mutation, editor mutation, terminal execution, or LSP authority.

<execution_contract>

- read-before-write: read `08-CONTEXT.md`, `08-03-RESULT.md`, `crates/devil-protocol/src/lib.rs`, `crates/devil-ui/src/ui.rs`, `crates/devil-app/src/lib.rs`, `crates/devil-desktop/src/bridge.rs`, `crates/devil-desktop/src/workflow.rs`, `crates/devil-desktop/src/view.rs`, `crates/devil-app/tests/workspace_vfs_integration.rs`, `plans/evidence/gui-productization/phase-8-collaboration-gui.md`, and `AGENTS.md` before editing.
- evidence-before-action: confirm `AppComposition` owns remote runtime enablement, remote session connection, remote session projections, and proposal-mediated remote write behavior before adding GUI controls.
- minimal diff: add only projection data, action routing, view rows, tests, evidence, and result content needed for remote GUI workflows.
- verify-before-report: run every frontmatter command and record outcomes in `08-04-RESULT.md`.
- role: remote GUI implementer and terminal/LSP boundary verifier.
- task: make remote workspace session/status workflows visible and usable from desktop projections.
- scope: remote connection manager rows, remote session status, terminal/LSP descriptors, reconnect/offline indicators, remote proposal review links, action validation, workflow outcomes, and evidence.
- allowed tools/actions: focused Rust/Markdown edits, targeted app/desktop tests, `rg`, and `xtask` dependency checks.
- forbidden actions: direct remote transport frame parsing in UI/desktop, direct local disk writes from remote GUI, direct editor mutation from remote GUI, terminal execution authority in UI/desktop, LSP authority in UI/desktop, raw remote payload logging, dependency additions, legacy Phase 8 evidence edits, and test weakening.
- verification criteria: remote runtime remains disabled by default; explicit remote enablement and connection produce projected rows; reconnect/offline status is metadata-only; remote mutation remains proposal-mediated; app remote tests still pass.
- final result format: write `.planning/phases/08-advanced-platform-gui-ga/08-04-RESULT.md` with changed files, command outcomes, and any `BLOCKED` items.

</execution_contract>

<stop_gates>

- BLOCKED if remote GUI status requires raw transport frames, raw payload bodies, shell output bodies, or file contents.
- BLOCKED if remote writes, remote terminal actions, or remote LSP actions would bypass app/proposal authority.
- BLOCKED if remote runtime must become enabled by default to make GUI workflows visible.
- BLOCKED if remote session projection cannot distinguish connected, reconnecting, offline, and denied states from safe metadata.
- BLOCKED if existing remote proposal-mediated mutation tests fail because this plan changed authority boundaries.

</stop_gates>

<recovery>

- If current remote session descriptors are not sufficient for GUI rows, add the smallest metadata-only projection DTOs and derive them in app composition.
- If remote connect cannot be represented by an existing UI intent, add a narrow `CommandDispatchIntent` or desktop app request that calls only app-owned remote connection methods.
- If remote proposal review already has sufficient proposal rows, add remote-context labels around existing proposal actions instead of duplicating proposal lifecycle controls.
- On interruption, inspect `git diff`, compare modified files against `files_modified` and `files_forbidden`, rerun targeted verification, and continue only from unverified work.

</recovery>
