# Plan 08-05: Delegated Task Command Center

## Objective

Expose a delegated task command center that shows plan rows, blockers, refusals, trust gates, proposal-preview links, and audit readiness while keeping delegated task execution bounded, approval-gated, metadata-only, and explicitly non-autonomous.

<execution_contract>

- read-before-write: read `08-CONTEXT.md`, `08-04-RESULT.md`, `crates/devil-protocol/src/lib.rs`, `crates/devil-ui/src/ui.rs`, `crates/devil-app/src/lib.rs`, `crates/devil-desktop/src/bridge.rs`, `crates/devil-desktop/src/workflow.rs`, `crates/devil-desktop/src/view.rs`, `crates/devil-desktop/src/health.rs`, `plans/evidence/gui-productization/phase-8-remote-workspace-gui.md`, and `AGENTS.md` before editing.
- evidence-before-action: confirm delegated task DTOs and helper projection functions already encode plan contracts and that runtime activation remains `NotEncoded` before adding GUI command-center rows.
- minimal diff: edit only app projection composition, desktop bridge/workflow/view command-center code, a focused desktop regression test file, evidence, and result content.
- verify-before-report: run every frontmatter command and record outcomes in `08-05-RESULT.md`.
- role: delegated task GUI implementer and autonomy-boundary verifier.
- task: make delegated task planning/review state usable from desktop projections without adding autonomous apply or agent runtime activation.
- scope: plan rows, blockers/refusals, trust gates, proposal-preview links, audit readiness, bridge validation, workflow outcomes, health labels, tests, and evidence.
- allowed tools/actions: focused Rust/Markdown edits, targeted desktop/UI tests, `rg`, and `xtask` dependency checks.
- forbidden actions: activating `devil-agent`, adding agent runtime execution, autonomous apply, direct file/editor mutation from delegated task rows, raw prompt/context persistence, dependency additions, legacy Phase 8 evidence edits, and test weakening.
- verification criteria: delegated task projection is app-composed and metadata-only; runtime activation remains not encoded; proposal-preview links route through existing proposal actions; blocked/refused plan states are visible; autonomous apply remains unsupported.
- final result format: write `.planning/phases/08-advanced-platform-gui-ga/08-05-RESULT.md` with changed files, command outcomes, and any `BLOCKED` items.

</execution_contract>

<stop_gates>

- BLOCKED if command-center implementation requires enabling autonomous agent execution.
- BLOCKED if delegated task GUI needs raw prompts, raw context manifests, dirty buffer text, or file contents in projections.
- BLOCKED if proposal-preview links cannot be validated against projected proposal ids.
- BLOCKED if health/status text would imply autonomous apply is supported.
- BLOCKED if placeholder crates need to become active without ADR/phase gate/dependency-policy entries and contract tests.

</stop_gates>

<recovery>

- If app composition currently hardcodes an empty delegated projection, add a test-only or app-owned setter for plan contract snapshots that remains metadata-only and does not activate runtime execution.
- If plan contract DTOs lack enough review metadata, derive command-center rows from existing safe fields and record any unavailable data explicitly in evidence.
- If proposal-preview routing is ambiguous, route only to existing proposal details/preview actions and deny unknown proposal links.
- On interruption, inspect `git diff`, compare modified files against `files_modified` and `files_forbidden`, rerun targeted verification, and continue only from unverified work.

</recovery>

- `crates/devil-ui/src/ui.rs`
- `crates/devil-app/src/lib.rs`

### Write Targets

- `crates/devil-app/src/lib.rs`

### Implementation Sequence

1. Replace any hardcoded empty delegated task projection in app snapshots with app-owned composition from safe plan contract data. Keep runtime activation set to `NotEncoded`.
   > verification: `rg -q "DelegatedTask" crates/devil-app/src/lib.rs`
2. Add a bounded app-owned way for tests/smoke harnesses to seed delegated task plan contracts without enabling an agent runtime.
   > verification: `cargo test -p devil-ui delegated -- --nocapture`
3. Ensure projected delegated task data contains only metadata-safe plan state, blocker/refusal labels, trust gate labels, proposal ids, and audit status.
   > verification: `rg -q "NotEncoded" crates/devil-app/src/lib.rs`

### Edge And Error Cases

- Reject or omit delegated task rows that lack stable plan ids.
- Do not project raw prompts, raw context, raw proposed file content, terminal output bodies, or provider payloads.
- Do not activate placeholder agent, memory, or tracker crates.
