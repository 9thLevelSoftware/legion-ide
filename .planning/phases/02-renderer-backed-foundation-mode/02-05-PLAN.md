# Plan 02-05: Renderer Timing And Platform Smoke Evidence

## Objective

Add automated renderer timing and platform smoke evidence so Phase 2 can prove the windowed shell behaves credibly before acceptance.

<execution_contract>

- read-before-write: read Phase 1 renderer readiness, ADR-0002 proof obligations, desktop workflow, and current metrics/smoke stubs before editing.
- evidence-before-action: record smoke evidence only from commands run in this checkout.
- minimal diff: edit desktop metrics/smoke/workflow integration and the smoke evidence file only.
- verify-before-report: run the platform smoke tests and the timed smoke command before marking the plan complete.
- role: act as performance and platform-verification engineer.
- task: produce p50/p95 input-to-paint, frame variance, focus, clipboard, IME, high-DPI, file-dialog, and accessibility smoke fields.
- scope: metadata-only timing/report structures, timed eframe close mode, synthetic bridge-level platform event smoke, and evidence writing.
- allowed tools/actions: use `std::time`, egui context metadata, desktop workflow harness, and the real `cargo run -p devil-desktop -- --smoke` command.
- forbidden actions: persisting raw source in metrics, claiming unsupported native behavior, adding new GUI dependencies, or weakening renderer gates.
- verification criteria: tests validate metric math and smoke report completeness; the timed smoke command writes evidence with all required fields.
- final result format: write plan result with metric values, smoke command output, and any platform limitations.

</execution_contract>

<stop_gates>

- BLOCKED if the timed smoke mode cannot open and close a native window in the current environment.
- BLOCKED if smoke evidence cannot distinguish passed, failed, and not-observed platform checks.
- BLOCKED if collecting metrics requires storing raw buffer text, clipboard contents, or private payloads.

</stop_gates>

<recovery>

- If the current environment cannot open a native window, keep unit-level metric/report tests passing, write the evidence file as `Phase 2 renderer smoke: Blocked`, and include the exact OS/runtime error.
- If clipboard or IME cannot be observed from eframe APIs, record bridge-level synthetic smoke as `adapter-path passed` and OS-observed smoke as `not observed` rather than claiming native proof.
- If work is interrupted, capture `git diff`, modified files, verification state, and `02-05-RESULT.md` status.

</recovery>

<verification>

Run the frontmatter verification commands. The plan is complete only when the evidence file exists and explicitly records every required proof field.

</verification>

<success_criteria>

- `FrameTimingRecorder` computes p50/p95 input-to-paint and frame variance deterministically.
- `--smoke` opens the desktop runtime, runs for a bounded duration, requests close, and writes metadata-only evidence.
- The smoke evidence names focus, clipboard, IME, high-DPI, file-dialog, and accessibility status without overstating unobserved platform behavior.

</success_criteria>

<output>

Write `.planning/phases/02-renderer-backed-foundation-mode/02-05-RESULT.md` with smoke status, exact metrics, verification commands, and residual platform risks.

</output>

## Task 1: Implement Timing Metrics

### Read Targets

- `plans/evidence/gui-productization/phase-1-renderer-readiness.md`
- `plans/adrs/ADR-0002-ui-editor-rendering.md`
- `crates/devil-desktop/src/metrics.rs`
- `crates/devil-desktop/src/workflow.rs`

### Write Targets

- `crates/devil-desktop/src/metrics.rs`

### Implementation Sequence

1. Define `InputPaintSample` with input timestamp, paint timestamp, and duration in milliseconds; do not include raw input text.
   > verification: `rg -q "InputPaintSample" crates/devil-desktop/src/metrics.rs`
2. Define `FrameTimingRecorder` with methods `record_input`, `record_paint`, `record_frame_duration`, and `summary`.
   > verification: `rg -q "FrameTimingRecorder" crates/devil-desktop/src/metrics.rs`
3. Define `FrameTimingSummary` with `sample_count`, `p50_input_to_paint_ms`, `p95_input_to_paint_ms`, `average_frame_ms`, and `frame_variance_ms2`.
   > verification: targeted tests assert exact summary values for deterministic samples.
4. Ensure all metrics are metadata-only and bounded in memory by retaining only durations and counters, not event payloads.
   > verification: inspect metrics structs; no text payload fields.

### Content Structure

- Keep metrics independent of egui so tests can run without a renderer context.
- Use `std::time::Instant` for live timing and plain `f64` millisecond values for summaries.
- Sort copied duration values inside summary calculation; do not mutate recorded order.

### Edge And Error Cases

- Empty recorder returns zero sample count and `None` or zero-valued percentile fields documented by tests.
- One-sample recorder returns the same value for p50 and p95.
- Paint without prior input records frame timing but not input-to-paint.

### Verification Commands

```powershell
rg -q "FrameTimingRecorder" crates/devil-desktop/src/metrics.rs
```

<verify>

Timing metrics are complete only when deterministic tests cover empty, one-sample, and multi-sample summaries.

</verify>

### BLOCKED Conditions

- Metrics require capturing user input contents or source text.

## Task 2: Add Timed Smoke Harness And Evidence Writer

### Read Targets

- `crates/devil-desktop/src/smoke.rs`
- `crates/devil-desktop/src/workflow.rs`
- `crates/devil-desktop/src/main.rs`
- `crates/devil-desktop/src/bridge.rs`
- `crates/devil-desktop/src/metrics.rs`

### Write Targets

- `crates/devil-desktop/src/smoke.rs`
- `crates/devil-desktop/src/workflow.rs`
- `crates/devil-desktop/src/main.rs`
- `plans/evidence/gui-productization/phase-2-renderer-smoke.md`

### Implementation Sequence

1. Add `DesktopLaunchConfig` parsing for `--smoke`, `--workspace <path>`, `--file <path>`, `--duration-ms <number>`, and `--evidence <path>`.
   > verification: smoke tests assert flag parsing.
2. Define `RendererSmokeReport` with fields for command, status, workspace, file, duration, timing summary, focus status, clipboard status, IME status, high-DPI status, file-dialog status, accessibility status, and errors.
   > verification: `rg -q "RendererSmokeReport" crates/devil-desktop/src/smoke.rs`
3. Implement timed smoke mode that creates the real `DesktopRuntime`, starts an eframe window, records frame/input metrics, and closes after the configured duration using the renderer close command.
   > verification: `cargo run -p devil-desktop -- --smoke --workspace . --file Cargo.toml --duration-ms 1500 --evidence plans/evidence/gui-productization/phase-2-renderer-smoke.md`
4. Write smoke evidence as markdown with stable keys including `p50_input_to_paint_ms`, `p95_input_to_paint_ms`, `frame_variance_ms2`, `focus_smoke`, `clipboard_smoke`, `ime_smoke`, `high_dpi_smoke`, `file_dialog_smoke`, and `accessibility_smoke`.
   > verification: `rg -q "accessibility_smoke" plans/evidence/gui-productization/phase-2-renderer-smoke.md`
5. If a platform check is synthetic or adapter-level rather than OS-observed, label it exactly as `adapter-path passed` or `not observed`; do not label it simply `passed`.
   > verification: evidence file contains no unsupported blanket claim.

### Content Structure

- `smoke.rs`: report structs, evidence markdown writer, timed smoke orchestration helpers.
- `workflow.rs`: smoke-aware launch config and timed close integration.
- Evidence markdown:
  - `# Phase 2 Renderer Smoke Evidence`
  - `## Status`
  - `## Command`
  - `## Timing`
  - `## Platform Smoke`
  - `## Errors`
  - `## Residual Risk`

### Edge And Error Cases

- If window creation fails, write a blocked evidence file with the exact error and return a non-zero process exit.
- If the smoke duration is zero, reject the config before opening a window.
- If evidence path parent directory is missing, create only that exact directory path, not broad project directories.

### Verification Commands

```powershell
cargo run -p devil-desktop -- --smoke --workspace . --file Cargo.toml --duration-ms 1500 --evidence plans/evidence/gui-productization/phase-2-renderer-smoke.md
rg -q "p95_input_to_paint_ms" plans/evidence/gui-productization/phase-2-renderer-smoke.md
rg -q "accessibility_smoke" plans/evidence/gui-productization/phase-2-renderer-smoke.md
```

<verify>

The smoke harness is complete only when it produces an evidence file from a real command and the file records all required proof fields.

</verify>

### BLOCKED Conditions

- The local environment cannot open an eframe window and no timed smoke evidence can be produced.

   > verification: same targeted test command.
3. Add flag parsing tests for `--smoke --workspace . --file Cargo.toml --duration-ms 1500 --evidence ...`.


<verify>

Platform smoke coverage is complete only when tests prove report completeness independently of whether the OS window smoke passes in the current environment.

</verify>
