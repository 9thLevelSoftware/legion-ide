# Plan 02-03: Desktop Intent Bridge And App Requests

## Objective

Translate renderer events into `CommandDispatchIntent` values or explicit app-owned requests without applying mutations inside the desktop adapter.

<execution_contract>

- read-before-write: read `CommandDispatchIntent`, `ShellProjectionSnapshot`, `AppComposition::dispatch_ui_intent`, ADR-0030, and Plan 02-01 stubs before editing.
- evidence-before-action: each bridge mapping must point to an existing app/UI intent or an explicit desktop app request consumed by Plan 02-04.
- minimal diff: edit only `bridge.rs` and its integration tests.
- verify-before-report: run targeted bridge tests and desktop package check.
- role: act as boundary engineer for renderer-to-app commands.
- task: create a small bridge that translates keyboard, menu, edit, save, open-path, path-dialog, and quit actions.
- scope: mapping from adapter-local `DesktopAction` to `DesktopBridgeOutput`; no app mutation execution.
- allowed tools/actions: create enums, pure mapping functions, tests with projection fixtures.
- forbidden actions: call `AppComposition`, write files, mutate editor buffers, manage proposals, or persist command state in the bridge.
- verification criteria: tests prove every Phase 2 user action becomes a `CommandDispatchIntent`, a `DesktopAppRequest`, a no-op cancellation, or a typed bridge error.
- final result format: write a plan result with mapping table and test output.

</execution_contract>

<stop_gates>

- BLOCKED if an action cannot be represented without inventing a new app mutation API.
- BLOCKED if mapping edit/save actions requires renderer-owned buffer or dirty state.
- BLOCKED if file-dialog cancellation would emit an app mutation.

</stop_gates>

<recovery>

- If an action needs a buffer id and the snapshot has no active buffer, return `DesktopBridgeError::MissingActiveBuffer` rather than creating a fake id.
- If text position cannot be inferred from the current projection, default only to an explicit cursor in adapter-local input state supplied by workflow; otherwise return a typed error.
- If work is interrupted, capture `git diff`, modified files, verification state, and `02-03-RESULT.md` status.

</recovery>

<verification>

Run the frontmatter verification commands. The bridge is complete only if tests cover happy paths, missing-buffer errors, dialog cancellation, and no direct app mutation.

</verification>

<success_criteria>

- Save, insert, replace, delete, open path, path-dialog selection, menu commands, keyboard shortcuts, and quit are represented without renderer-owned authority.
- The bridge never calls workspace/editor/storage/proposal APIs.
- Tests fail if a missing buffer silently emits a mutation intent.

</success_criteria>

<output>

Write `.planning/phases/02-renderer-backed-foundation-mode/02-03-RESULT.md` with supported actions, error cases, and verification output.

</output>

## Task 1: Define Desktop Action And Request Types

### Read Targets

- `crates/devil-ui/src/ui.rs`
- `plans/desktop-adapter-boundary-v0.1.md`
- `crates/devil-desktop/src/bridge.rs`

### Write Targets

- `crates/devil-desktop/src/bridge.rs`

### Implementation Sequence

1. Define `DesktopAction` variants for `Quit`, `SaveActive`, `OpenPathText(String)`, `OpenPathDialogSelected(String)`, `OpenPathDialogCancelled`, `RefreshExplorer`, `InsertText { text, at }`, `ReplaceRange { range, replacement }`, `DeleteRange { range }`, `Undo`, and `Redo`.
   > verification: `rg -q "enum DesktopAction" crates/devil-desktop/src/bridge.rs`
2. Define `DesktopAppRequest` variants for app-level requests that are not direct `CommandDispatchIntent`, including `OpenWorkspace { root }` and `ShowOpenPathPrompt`.
   > verification: `rg -q "enum DesktopAppRequest" crates/devil-desktop/src/bridge.rs`
3. Define `DesktopBridgeOutput` as one of `Intent(CommandDispatchIntent)`, `AppRequest(DesktopAppRequest)`, `Noop`, or `Error(DesktopBridgeError)`.
   > verification: `rg -q "DesktopBridgeOutput" crates/devil-desktop/src/bridge.rs`
4. Define typed errors `MissingActiveBuffer`, `InvalidPathInput`, and `UnsupportedAction`.
   > verification: `rg -q "MissingActiveBuffer" crates/devil-desktop/src/bridge.rs`

### Content Structure

- Keep all bridge types in `bridge.rs`.
- Use `std::path::PathBuf` only for explicit app requests, not for direct filesystem mutation.
- Use `devil_protocol::TextCoordinate` and `devil_protocol::ProtocolTextRange` for edit locations.

### Edge And Error Cases

- Empty path text returns `InvalidPathInput`.
- Dialog cancellation returns `Noop`.
- Missing active buffer returns `MissingActiveBuffer` for save/edit/undo/redo.

### Verification Commands

```powershell
rg -q "DesktopAppRequest" crates/devil-desktop/src/bridge.rs
```

<verify>

The type surface is complete only when every required Phase 2 action has an explicit variant and no variant carries editor/workspace internals.

</verify>

### BLOCKED Conditions

- Required actions cannot be represented without adding new public protocol types.

## Task 2: Implement Pure Intent Mapping

### Read Targets

- `crates/devil-ui/src/ui.rs`
- `crates/devil-desktop/src/bridge.rs`
- `crates/devil-desktop/src/view.rs`

### Write Targets

- `crates/devil-desktop/src/bridge.rs`

### Implementation Sequence

1. Implement `DesktopCommandBridge::translate(action: DesktopAction, snapshot: &ShellProjectionSnapshot) -> DesktopBridgeOutput`.
   > verification: `rg -q "fn translate" crates/devil-desktop/src/bridge.rs`
2. For save/undo/redo/edit variants, read `snapshot.active_buffer_projection.buffer_id` and emit the corresponding `CommandDispatchIntent` only when it exists.
   > verification: targeted tests assert matching buffer ids.
3. For open path text or dialog selection, trim path input and emit `CommandDispatchIntent::OpenPath { path }`; cancellation emits `Noop`.
   > verification: targeted tests assert cancellation does not create an intent.
4. For `Quit`, emit `CommandDispatchIntent::Quit`; for `ShowOpenPathPrompt`, emit `DesktopAppRequest::ShowOpenPathPrompt`.
   > verification: targeted tests assert output variants.
5. Keep proposal/plugin/AI/collaboration commands out of Phase 2 bridge unless already selected from projection data by later UI controls; do not invent runtime behavior.
   > verification: no app/proposal coordinator imports in `bridge.rs`.

### Content Structure

- One helper for active buffer lookup.
- One helper for path input validation.
- No global mutable state.

### Edge And Error Cases

- Preserve Unicode text in insert/replace payloads.
- Do not clamp invalid protocol ranges silently; bridge should pass well-formed ranges supplied by workflow or return `UnsupportedAction` if the workflow cannot produce one.
- Do not transform paths into canonical paths; app/workspace authority owns path resolution.

### Verification Commands

```powershell
cargo check -p devil-desktop --all-targets
```

<verify>

Pure mapping is complete only when `bridge.rs` imports no app/editor/workspace/storage modules and cannot mutate product state by construction.

</verify>

### BLOCKED Conditions

- Mapping requires app-private state not present in `ShellProjectionSnapshot`.

## Task 3: Add Bridge Regression Tests

### Read Targets

- `crates/devil-desktop/src/bridge.rs`
- `crates/devil-ui/src/ui.rs`
- `crates/devil-protocol/src/lib.rs`

### Write Targets

- `crates/devil-desktop/tests/intent_bridge.rs`

### Implementation Sequence

1. Add fixture helpers for snapshots with and without active `BufferId`.
   > verification: tests compile without `devil-app` imports.
2. Add tests for save, insert, replace, delete, undo, redo, open path, dialog selection, dialog cancellation, prompt request, and quit.
   > verification: `cargo test -p devil-desktop intent_bridge --test intent_bridge`
3. Add tests for missing active buffer errors on edit/save/undo/redo actions.
   > verification: same targeted test command.
4. Add a source-level boundary assertion in the test comments or assertions that `bridge.rs` has no `AppComposition`, `WorkspaceActor`, or `EditorEngine` imports.
   > verification: `if (rg "AppComposition|WorkspaceActor|EditorEngine" crates/devil-desktop/src/bridge.rs) { exit 1 }` returns success only when there are no matches.

### Content Structure

- Keep each test focused on one action class.
- Assert exact `CommandDispatchIntent` values where equality is available.
- Assert exact bridge error variants for failure paths.

### Edge And Error Cases

- Empty and whitespace-only paths must be rejected.
- Dialog cancellation must not emit `OpenPath`.
- Unicode edit text must be preserved exactly.

### Verification Commands

```powershell
cargo test -p devil-desktop intent_bridge --test intent_bridge
cargo check -p devil-desktop --all-targets
```

<verify>

Bridge coverage is complete only when tests cover every Phase 2 action route and every bridge error listed in Task 1.

</verify>

### BLOCKED Conditions

- Public projection fixtures cannot be created without app-private state.
