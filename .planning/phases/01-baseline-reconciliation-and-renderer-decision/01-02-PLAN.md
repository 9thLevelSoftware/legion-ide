# Plan 01-02: Renderer Decision ADR And Matrix

## Objective

Reconfirm the renderer decision against current primary documentation and capture the evidence needed to choose the Phase 2 implementation path.

<execution_contract>

- read-before-write: read ADR-0002, SPIKE-001A, dependency policy, crate manifests, and current primary renderer/accessibility documentation before editing.
- evidence-before-action: record source URLs and repository facts in the matrix before changing the ADR.
- minimal diff: update only ADR-0002 decision details and add the renderer decision matrix.
- verify-before-report: run every verification command and do not report success if the renderer decision remains ambiguous without a `BLOCKED` reason.

</execution_contract>

<stop_gates>

- BLOCKED if official documentation cannot be checked for the selected renderer candidate, accessibility path, and Windows desktop viability.
- BLOCKED if no candidate can satisfy the minimum Phase 2 criteria: Windows-first desktop shell, controlled text rendering path, key input, IME, clipboard, focus, accessibility story, high DPI, file dialogs or native bridge, input-to-paint measurement, and projection-only compatibility.
- BLOCKED if the ADR would authorize renderer dependencies in `devil-ui`.

</stop_gates>

<recovery>

- If a preferred candidate has a missing capability, define a fallback trigger and evidence requirement rather than hand-waving the gap.
- If documentation is unavailable, keep ADR-0002 at the existing accepted-with-reservations state and write a `BLOCKED` matrix section with exact missing sources.
- If a candidate requires a production dependency that is not policy-authorized, route that requirement into plan 01-04 instead of editing manifests here.

</recovery>
